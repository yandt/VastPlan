## 顶层树

```
VastPlan/
├── docs/dev/                      # 设计文档（架构 / ADR / 指南 / 插件文档）
├── schemas/                        # 机器可执行 JSON Schema（唯一契约源）
│   ├── common/v1/                  #   公共标识规则与跨版本稳定资源 DTO
│   ├── plugin/v1/                  #   插件清单 / descriptor / 制品元数据
│   └── deployment/                  #   v1 节点执行快照 + v2 集群部署规格
├── proto/                         # 契约与协议的单一真源（Protobuf）
│   ├── contract/v1/               #   CallContext / CallTarget / CallResult / CallEvent
│   ├── pluginhost/v1/             #   宿主↔插件协议（Hello / Register / Channel）
│   └── addressing/v1/             #   跨服务调用/事件 wire 信封
├── kernels/                       # 四内核（规范 ID：backend/frontend/runner/mobile）
│   ├── backend/                   #   Go：backend 宿主工厂 · 节点代理 · 制品信任/种子基座
│   │   ├── commands/              #     生产子命令组合根（controlplane / artifact-server）
│   │   ├── hostfactory/           #     手工启动与自动装配共用的扩展点/内置能力
│   │   ├── nodeagent/             #     DesiredState reconcile / 安装 / 运行时 / 实际态
│   │   ├── deploymentcontroller/  #     Deployment v2 → 每节点 assignment v1 调度
│   │   └── pluginservice/         #     本地/HTTPS 制品源与迁移期 artifact-server 实现
│   ├── frontend/                  #   TS/React：Module Federation 宿主 · slot 装配
│   ├── runner/                    #   Go：客户端执行器 · Launcher/自更新 · 内容脚本运行时
│   └── mobile/                    #   原生外壳(iOS/Android) + gomobile bind 的 Go 核心
├── internal/                      # 仓内多入口复用、但不对外承诺兼容的窄领域包
│   └── soakreport/                #   soak 报告结构与离线验收规则（当前不执行 soak）
├── shared/                        # 共享库
│   ├── go/
│   │   ├── artifacttrust/          #   来源不可绕过的制品内容验证与 Envelope 契约
│   │   ├── contract/              #   proto 生成码 + 契约辅助
│   │   ├── registry/              #   扩展点注册表
│   │   ├── protocolbus/           #   宿主↔插件协议总线（gRPC）
│   │   ├── addressing/            #   本地直调 + NATS request-reply 寻址
│   │   ├── controlplane/          #   NATS KV/CAS/租约与稳定 key/subject
│   │   └── clientcore/            #   客户端共享核心（仅 runner + mobile 共用）
│   └── ts/contract/               #   前端契约类型（proto 生成）
├── plugins/                       # 第一方插件：一插件一目录，四面同处
│   └── <plugin-id>/
│       ├── vastplan.plugin.json   #   清单（四面贡献声明）
│       ├── backend/               #   backend 面（Go 或 Python）
│       ├── frontend/              #   frontend 面（TS/React）
│       ├── runner/                #   runner 面（脚本/工作流内容；编译型插件源码）
│       └── mobile/                #   mobile 面
├── sdk/                           # 插件开发 SDK
│   ├── go/                        #   Go SDK：协议、生命周期、动态贡献、事件、取消
│   ├── python/                    #   Python SDK + proto 生成物（经 gRPC 接入，ADR-0009/0047）
│   └── ts/                        #   前端插件
├── e2e/                           # 跨组件 E2E（build tag `e2e`，见测试规范）
│   └── fixtures/plugins/          #   夹具插件（故意异常，不进 plugins/）
├── deploy/                        # 期望态样例 · 环境 profile · 编排
└── tools/                         # codegen · 构建 · 测试 · 发布脚本
```

> 单元测试**与源码同目录**（不建统一 tests/ 树）；跨组件 E2E 才进 `e2e/`。详见《[测试规范](测试规范.md)》。

## 各目录职责

| 目录 | 职责 | 不该放什么 |
|---|---|---|
| `proto/` | 契约与协议的**唯一**定义源 | 手写的重复契约结构（一律 codegen） |
| `schemas/*/vN/` | 公共标识、插件与部署 JSON Schema 及 Go 校验入口 | 手写的第二套契约规则 |
| `kernels/*` | 各内核的**最小骨架**：宿主、注册表、协议总线、横切、生命周期 | **业务功能**（一律下沉为插件） |
| `shared/go/*` | 跨内核复用的通用能力 | 只被单一内核用的代码（放该内核下） |
| `shared/go/clientcore` | Runner 与 Mobile **共用**的客户端核心 | 后端专用逻辑 |
| `plugins/<id>/*` | 一个第一方插件的全部面 | 内核能力、跨插件公共代码（提到 shared） |
| `sdk/*` | 插件开发者用的 SDK 与脚手架 | 内核内部实现细节 |
| `deploy/` | 期望态样例、环境 profile | 代码 |
| `tools/` | codegen / 构建 / 发布 | 运行时代码 |

## 关键约定

1. **服务组合是配置，不是代码**（ADR-0016 §3）：backend / workspace / rs 是**同一个 `kernels/backend` 二进制**部署多次 + 期望态 `service_role` 不同。Controller 仍由 `controlplane` 子命令启动；`artifact-server` 是 ADR-0049 迁移期兼容入口，仓库服务最终由预置基础插件承载。
2. **内核不含业务**：凡"换一种实现是合理需求"的能力（审计、可观测、权限策略、用户校验）都放 `plugins/`，内核只留扩展点与契约。
3. **唯一的编译期例外已收窄**：内核只保留制品信任、种子源、可信缓存和安装强制点；仓库 HTTP/存储/索引/市场能力进入预置基础插件（ADR-0049）。
4. **proto 改动流程**：改 `proto/` → 跑 `tools/gen-proto.sh` → 同一次提交更新 `shared/go/*`、`sdk/python/{contract,pluginhost}` 与受影响 SDK；不得手写重复消息类型。
5. **插件面目录名 = 规范 ID**（ADR-0015）：`backend/` `frontend/` `runner/` `mobile/`，与清单 `contributes.<id>` 一致。
6. **clientcore 边界**：只放 Runner 与 Mobile 都要的东西（契约、鉴权、清单模型、配置）；任一方专用的不进去。
7. **Backend 横向依赖**：普通子包通过 `schemas/` DTO 或窄接口协作，不引用同级实现包；只有 `kernels/backend` 主包和 `commands/` 组合根可以装配多个同级组件。
8. **版本 DTO 单一真源**：跨 schema 版本完全同语义的 Go DTO 放 `schemas/common/v1`，版本包只留别名；外部 JSON 语义不同的类型不得强行合并，必须显式转换。

## "我要加东西放哪"决策树

- 加一个**业务功能** → `plugins/<新插件>/`，按面建子目录。
- 加一个**扩展点/内核能力** → 对应 `kernels/<内核>/`，并在《插件契约与协议》第四章登记契约。
- 加一个**跨内核复用的通用能力** → `shared/go/<模块>`。
- 加一个**契约字段/协议消息** → `proto/`，然后 codegen（勿手写）。
- 加一个**插件 JSON 清单/descriptor/制品字段** → `schemas/plugin/v1/`；改节点执行快照 → `schemas/deployment/v1/`；改全局副本/放置 → `schemas/deployment/v2/`；先改 JSON Schema，再让各语言消费它。
- 加一个**后端服务**（如新 service_role） → **不加代码**，改 `deploy/` 期望态即可。
