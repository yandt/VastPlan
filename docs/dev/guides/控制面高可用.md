# 控制面高可用运行指南

Controller 可以按同一个 Deployment key 启动多个副本。每个副本必须使用独立 NKey、mTLS 客户端证书和 `controller-id`。

```bash
go run ./core/kernels/backend controlplane -controller \
  -controller-id controller-a \
  -repository /var/lib/vastplan/repository \
  -nats-url tls://nats.example.com:4222 \
  -nats-ca /etc/vastplan/pki/ca.crt \
  -nats-cert /etc/vastplan/pki/controller-a.crt \
  -nats-key /etc/vastplan/pki/controller-a.key \
  -nats-seed /secure/controller-a.seed \
  -key tenants.X2dsb2JhbA.states.Y2x1c3Rlcg
```

其他副本只更换 identity、证书与 seed。日志出现“获得领导权”后该副本才会启动 Deployment/Node watcher；失去领导权时会取消在途调度并退回候选状态。

## 故障语义

- 正常退出：使用 KV revision 条件删除领导权，备用副本通常在下一次重试周期接管。
- 进程崩溃或网络隔离：续租停止，记录过期后备用副本 CAS 接管。
- 旧 Leader 恢复：旧 revision/token 无法删除新 Leader；它重新作为候选者等待。
- Assignment 部分写入：NATS KV 没有跨 key 事务；新 Leader 根据完整 Deployment 和节点租约重新计算，以新 generation 覆盖并清理旧节点 Assignment。

## Drain 语义

Runtime 替换 unit 时先发布候选能力，再摘除旧能力租约，然后进入宿主 Drain：

1. 原子拒绝旧宿主的新 Invoke/Event。
2. 等待已经进入完整 Hook/权限/分发管线的调用结束。
3. 向旧插件发送 DRAIN 生命周期指令。
4. SHUTDOWN、摘除贡献并回收进程。

如果 Drain 超时，上层会记录错误并强制回收；调用方应使用 request ID/业务幂等键处理边界重试。
