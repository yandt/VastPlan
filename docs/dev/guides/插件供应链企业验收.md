# 插件供应链企业验收

本文给出 ADR-0135—0138 的上线验收入口，覆盖插件 SBOM、来源证明、安全准入、持续复扫、stable 审批、气隙 Bundle 和 Node 二次验证。它不替代企业自己的安全扫描策略，也不包含此前明确延期的 24 小时 soak。

## 1. 验收边界

验收分成两层：

- **仓库自动化证据**：任何开发机都能重复执行，用于证明协议、签名、摘要、状态机和防回滚实现没有回归；
- **目标企业环境证据**：必须使用企业真实 CA、Secret/Vault、扫描 Provider、漏洞数据库镜像、网络代理、气隙介质和运维身份执行，仓库内测试不能冒充。

不以“页面显示绿色”代替原始签名记录，不以 Backend 源码 CI 的 govulncheck/license 检查代替插件制品扫描，也不以单次发布成功证明持续复扫调度有效。

## 2. 上线前置条件

1. 发布者、来源证明 Verifier、安全评估 Provider 使用三组不同 Ed25519 key；私钥权限不宽于 `0600`，分别进入构建、来源验证和安全扫描工作负载。
2. 仓库读取、发布、Bundle、Assessment 使用四个不同 token；Portal、普通发布流水线和扫描 Worker 不得互相取得对方 token。
3. 生产信任文档为 `stable` 配置 `provenance.requiredChannels` 与 `assessment.requiredChannels`，并为 testing/stable 配置相同 policy ID 以支持不可变 sidecar 晋级。
4. 扫描 Provider 固定报告 scanner/version/database revision，漏洞和许可证原始报告进入企业不可变归档；VastPlan 记录只保存其 SHA-256。
5. 所有 Node、Portal 物化宿主、Runner/Mobile 未来接入点分发同一代信任快照；过期或撤销 key 的处理流程已明确。
6. 复扫周期、记录 TTL 和告警窗口满足企业 SLA，并预留扫描服务不可用时的 fail-closed 处置流程。

## 3. 自动化有界故障矩阵

```bash
./engineering/tools/artifact-security-fault-matrix.sh
```

该脚本验证：不可变数据库快照拒绝摘要漂移与符号链接；报告归档拒绝篡改和宽权限；普通制品 Ticket 与独立报告 Ticket 不能跨路径使用；准入记录绑定 tar/SBOM；阈值、过期和签名错误拒绝；复扫 pending、失败重试与数据库 revision 切换收敛；失败记录可保存并阻止交付；更高 sequence 的通过记录可恢复；旧链不能越过 Node 本地高水位；Assessment token 不能读取、读 token 不能写；stable 审批绑定 sidecar；离线 Bundle 保留报告与状态链；Portal 评估 revision、权限和报告预授权路由可用。脚本还运行快照与 Controller 的短时真实插件进程 E2E，只使用临时数据，不连接既有仓库，不执行 soak。

## 4. 目标企业环境矩阵

| 场景 | 操作 | 必须观察到的结果 | 保存的证据 |
|---|---|---|---|
| 正常 testing 发布 | 用真实 Provider 生成 SBOM、Provenance、Admission 后发布 | 仓库与 Node 均复验，Catalog 显示 scanner/database revision | tar、三份 sidecar、原始报告、Catalog 响应 |
| stable 双人审批 | A 提交、B 批准，CI 原样复用 testing 包 | 同人批准拒绝；替换任一 sidecar 拒绝 | 审批记录、最终证明摘要、审计日志 |
| critical/high 超阈值 | Provider 签发 fail 或超阈值记录 | 发布或新安装 fail-closed，不产生可交付 stable | 失败记录、仓库审计、Node ActualState 错误 |
| 拒绝/未知许可证 | 报告包含策略禁止或未知许可证 | 与漏洞超阈值相同，不允许靠修改 Portal 状态放行 | 许可证报告及摘要、策略 revision |
| 数据库更新发现新漏洞 | 对不变 tar 追加 fail 状态 | fail 状态成功落盘；新交付阻止；既有进程按应急策略处置 | 状态 sequence/previous digest、告警时间 |
| 修复或例外策略通过 | 追加更高 sequence 的 pass 状态 | 链不断裂，新交付恢复；旧 fail 仍可审计 | 完整状态链、批准的例外/修复依据 |
| 仓库返回旧链 | 在隔离测试代理中截短链尾 | 已见更高 sequence 的 Node 因本机高水位拒绝 | Node 错误、watermark 摘要、代理记录 |
| Assessment token 越权 | 用 scanner token 读制品/Bundle，或用读写 token 追加状态 | 全部返回未授权，不泄漏其他 token | HTTP 状态与脱敏审计日志 |
| Provider key 撤销 | 更新信任快照撤销评估 key | 新发布、复扫与新安装拒绝；旧记录不被“回填时间”复活 | 信任快照 revision、拒绝日志 |
| 记录/报告被篡改 | 改一字节 Admission、Status 或归档报告 | sidecar 签名/摘要复验失败；归档对账失败 | 被篡改样本摘要、错误代码 |
| 报告下载权限旁路 | 用普通 `artifact-data` Ticket 请求报告，或用报告 Ticket 请求制品 | 两种跨路径安装均拒绝；报告还需独立权限与当前证据引用 | Exposure、Ticket 安装拒绝和脱敏审计 |
| 气隙 Bundle | 在线导出后在隔离区导入 | 包、证明、准入和复扫状态完整；隔离区再次复验 | Bundle SHA、导入清单、隔离区日志 |
| File Volume 迁移中复扫 | observing 阶段追加状态并故障注入一侧 | 双写不一致冻结并告警；重试同记录幂等收敛 | 两卷链头、迁移错误、恢复记录 |
| 扫描 Provider/数据库不可用 | 停止 Provider 或冻结数据库更新超过 TTL | 状态进入 stale 告警，新交付 fail-closed；仓库其他读功能可诊断 | 告警时间线、TTL、恢复时间 |

## 5. 通过标准

- 所有自动化门禁为绿色，且目标环境矩阵逐项有时间、操作者、环境、策略 revision、制品 SHA 和原始证据位置；
- `securityAssessment.rescanFailed/stale/invalid` 能被企业监控采集，出现任一项时告警；不得把插件 ID 作为指标 label；
- 从新漏洞进入数据库到仓库 fail 状态可见的时间不超过企业复扫 SLA；从恢复 pass 到 Node 可重新交付的时间有实测值；
- 日志、Portal、支持包和告警中没有 token、私钥、Credential material、原始 SBOM/扫描报告或包正文；
- 任何失败都不能通过覆盖同版本、替换 sidecar、缩短状态链或使用其他 token 绕过。

验收记录应进入企业自己的变更/审计系统；本仓库只保存可执行模板与不含生产秘密的结果摘要。
