# 测试规范

> 状态：v0.1（随代码演进）｜最后更新：2026-07-15
> 决策与理由见《[ADR-0018 测试布局与分层](../decisions/ADR-0018-测试布局与分层.md)》。
> 本文是**测试放哪、怎么写、怎么跑**的单一真相源。

## 一、放哪（位置规则）

| 层 | 位置 | 命名 / 包 | build tag | 何时跑 |
|---|---|---|---|---|
| **单元测试** | 与源码**同目录** | Go `xxx_test.go`（`package foo`，可测私有）<br>TS `xxx.test.ts` 就近 | 无 | 每次 `go test ./...` |
| **包黑盒测试** | 同目录 | Go `package foo_test`（只经导出 API） | 无 | 同上 |
| **E2E / 跨组件** | 顶层 `e2e/` | `e2e/<场景>_test.go` | `//go:build e2e` | 按需 / CI 单独阶段 |
| **数据夹具** | 就近 `testdata/` | — | — | 被测试读取 |
| **夹具插件** | `e2e/fixtures/plugins/<name>/` | — | — | E2E 内构建 |

**核心规则**：
1. **按语言惯例就近放，不建统一 `tests/` 树**（ADR-0018 §1）——统一的是分层与命名，不是物理位置。
2. **跨组件 E2E 才进 `e2e/`**，且必须加 `//go:build e2e`，避免慢测拖累日常。
3. **夹具插件不进 `plugins/`**——那里是产品插件（ADR-0016 §5）。为测异常路径而生的插件（故意崩溃、engines 不符、重复能力）一律放 `e2e/fixtures/plugins/`。
4. **E2E 不 mock 契约**——走真实 proto/协议/进程，否则测不出协议漂移。

## 二、怎么写

- **命名**：`Test<被测对象>_<场景>`，如 `TestRegister_SingleDispatchRejectsDuplicate`。
- **表驱动优先**：多分支用 `cases := []struct{...}` + `t.Run(c.name, ...)`。
- **断言消息说清"期望什么、实际什么"**：`t.Fatalf("期望按 priority 降序 [high low]，实际 [%s %s]", ...)`。读失败信息就能定位，不必回看代码。
- **注释写"为什么这条不变量重要"**，而非复述代码：
  ```go
  // single 语义下同一能力 id 不得被两个插件重复提供。
  func TestRegister_SingleDispatchRejectsDuplicate(t *testing.T) { ... }
  ```
- **fail-closed 路径必须有测试**（ADR-0017 §4 的五个强制点）——只测通过路径等于没测。

## 三、怎么跑

```bash
./tools/test.sh          # 单元测试（快，日常）
./tools/test.sh --e2e    # 单元 + E2E（含跨进程真实链路）
```

等价于：

```bash
go test ./...                    # 单元（e2e 因 build tag 被排除）
go test -tags=e2e ./e2e/...      # E2E
```

## 四、当前测试清单

| 位置 | 覆盖 |
|---|---|
| `shared/go/registry/registry_test.go` | 扩展点注册表：未定义扩展点拒绝、single 重复 id 拒绝、fanout 按 priority 排序、Lookup、插件崩溃批量摘除、解绑后可重注册（热装） |
