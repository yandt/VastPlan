# 测试规范

> 状态：v0.1（随代码演进）｜最后更新：2026-07-15
> 决策与理由见《[ADR-0018 测试布局与分层](../decisions/ADR-0018-测试布局与分层.md)》。
> 本文是**测试放哪、怎么写、怎么跑**的单一真相源。

## 一、放哪（位置规则）

| 层 | 位置 | 命名 / 包 | build tag | 何时跑 |
|---|---|---|---|---|
| **单元测试** | 与源码**同目录** | Go `xxx_test.go`（`package foo`，可测私有）<br>TS `xxx.test.ts` 就近 | 无 | 每次 `go test ./...` |
| **包黑盒测试** | 同目录 | Go `package foo_test`（只经导出 API） | 无 | 同上 |
| **架构守护** | 顶层 `arch/` | `arch/arch_test.go` | 无（快，必须每次跑） | 每次 `go test ./...` |
| **E2E / 跨组件** | 顶层 `e2e/` | `e2e/<场景>_test.go` | `//go:build e2e` | 按需 / CI 单独阶段 |
| **数据夹具** | 就近 `testdata/` | — | — | 被测试读取 |
| **夹具插件** | `e2e/fixtures/plugins/<name>/` | — | — | E2E 内构建 |

> **架构守护**（`arch/`）把文档里的架构约束变成可执行断言（依赖方向、单一真源、布局纪律、文档死链）。
> 它不加 build tag——只解析 import 与扫文件，很快，且违规必须**立即**暴露。详见《[工程规范](工程规范.md)》§5/§6。

**核心规则**：
1. **按语言惯例就近放，不建统一 `tests/` 树**（ADR-0018 §1）——统一的是分层与命名，不是物理位置。
2. **跨组件 E2E 才进 `e2e/`**，且必须加 `//go:build e2e`，避免慢测拖累日常。
3. **夹具插件不进 `plugins/`**——那里是产品插件（ADR-0016 §5）。为测异常路径而生的插件（故意崩溃、engines 不符、重复能力）一律放 `e2e/fixtures/plugins/`。
4. **E2E 不 mock 契约**——走真实 proto/协议/进程，否则测不出协议漂移。

## 二、怎么写

- **命名**：`Test<被测对象>_<场景>`，如 `TestRegister_SingleDispatchRejectsDuplicate`。
- **表驱动优先**：多分支用 `cases := []struct{...}` + `t.Run(c.name, ...)`。
- **断言消息说清"期望什么、实际什么"**：`t.Fatalf("期望按 priority 降序 [high low]，实际 [%s %s]", ...)`。读失败信息就能定位，不必回看代码。
- **注释写"为什么这条不变量重要"**，而非复述代码：
  ```go
  // single 语义下同一能力 id 不得被两个插件重复提供。
  func TestRegister_SingleDispatchRejectsDuplicate(t *testing.T) { ... }
  ```
- **fail-closed 路径必须有测试**（ADR-0017 §4 的五个强制点）——只测通过路径等于没测。

## 三、怎么跑

```bash
./tools/test.sh          # 单元测试（快，日常）
./tools/test.sh --e2e    # 单元 + E2E（含跨进程真实链路）
```

等价于：

```bash
go test ./...                    # 单元（e2e 因 build tag 被排除）
go test -tags=e2e ./e2e/...      # E2E
```

## 四、当前测试清单

| 位置 | 覆盖 |
|---|---|
| `shared/go/registry/registry_test.go` | 扩展点注册表：未定义扩展点拒绝、single 重复 id 拒绝、fanout 按 priority 排序、Lookup、插件崩溃批量摘除、解绑后可重注册（热装） |
| `shared/go/protocol/protocol_test.go` | 协议版本协商（含无交集）、engines SemVer 范围校验（含未声明→拒绝）、Supports |
| `shared/go/protocolbus/host_internal_test.go` | **同包私有逻辑**：双向流请求/响应关联（含错投、迟到、release 后投递）、**插件崩溃唤醒所有在途等待者**、markDead 幂等、request_id/票据唯一性、心跳闲置判据、默认时限合理性 |
| `e2e/plugin_lifecycle_test.go` | 跨进程真实链路（7 例）：拉起→回连→握手→engines 校验→贡献注册→激活→调用→摘除；**插件回调宿主**（含反方向 CallContext 透传）；应用层错误不冒泡为传输层错误；未注册能力拒绝；**内核版本不兼容/未声明 engines 拒绝装载**；**插件崩溃后贡献被摘除且在途调用立刻脱身** |
| `arch/arch_test.go` | **架构守护**：依赖方向（内核↛插件、共享↛组件、SDK↛内核、插件间不互依、clientcore 仅 Runner/Mobile）、单一真源（协议常量、契约结构不得手写）、布局纪律（无 services/ 目录、插件必有清单）、文档无死链 |
| `arch/design_test.go` | **代码设计守护**（ADR-0020）：禁垃圾抽屉包名（util/common/base…）、每个包必须有 doc comment（内聚试金石） |
| `e2e/dispatch_test.go` | **分发语义**（§4.2/ADR-0021）：select——零校验器 fail-closed、基线放行、**高优先级 deny 胜过低优先级 allow**、abstain 落到下一个、全弃权拒绝；fanout——事件送达订阅者（查审计插件账本验证真送达）、未订阅类型不投递、subscribe glob 匹配 |
| `shared/go/addressing/router_test.go` | NATS mesh：本地零序列化直调、远端 request-reply、queue group、错误分层、取消传播、非持久事件与租约撤销 |
| `shared/go/addressing/stream_test.go` | gRPC 双向流：输入/输出顺序、最终结果、上下文透传、远端取消、生产 mTLS fail-closed |
| `shared/go/addressing/persistent_test.go` | JetStream 持久事件：至少一次重投、严格顺序、durable 离线续消费、事件 ID 去重 |
| `kernels/backend/deploymentcontroller/scheduler_test.go` | 多节点调度：标签过滤、扩缩容、rendezvous 放置、容量不足 fail-closed、节点漂移与 KV watch |
| `e2e/node_reconcile_test.go` | Node Agent：真实进程幂等/回滚/崩溃恢复、NATS KV 触发、Runtime capability 发布、三节点两副本放置与故障接管 |
