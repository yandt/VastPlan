# 测试规范

> 状态：v0.1（随代码演进）｜最后更新：2026-07-18
> 决策与理由见《[ADR-0018 测试布局与分层](../decisions/ADR-0018-测试布局与分层.md)》。
> 本文是**测试放哪、怎么写、怎么跑**的单一真相源。

## 一、放哪（位置规则）

| 层 | 位置 | 命名 / 包 | build tag | 何时跑 |
|---|---|---|---|---|
| **单元测试** | 与源码**同目录** | Go `xxx_test.go`（`package foo`，可测私有）<br>TS/TSX `xxx.test.ts(x)` 就近<br>Python `test_*.py` 就近或 SDK `tests/` | 无 | 统一测试入口 / CI |
| **包黑盒测试** | 同目录 | Go `package foo_test`（只经导出 API） | 无 | 同上 |
| **架构守护** | 顶层 `engineering/arch/` | `engineering/arch/arch_test.go` | 无（快，必须每次跑） | 每次 `go test ./...` |
| **E2E / 跨组件** | 顶层 `engineering/e2e/` | `engineering/e2e/<场景>_test.go` | `//go:build e2e` | 按需 / CI 单独阶段 |
| **数据夹具** | 就近 `testdata/` | — | — | 被测试读取 |
| **夹具插件** | `engineering/e2e/fixtures/plugins/<name>/` | — | — | E2E 内构建 |

> **架构守护**（`engineering/arch/`）把文档里的架构约束变成可执行断言（依赖方向、单一真源、布局纪律、文档死链）。
> 它不加 build tag——只解析 import 与扫文件，很快，且违规必须**立即**暴露。详见《[工程规范](工程规范.md)》§5/§6。

**核心规则**：
1. **按语言惯例就近放，不建统一 `tests/` 树**（ADR-0018 §1）——统一的是分层与命名，不是物理位置。
2. **跨组件 E2E 才进 `engineering/e2e/`**，且必须加 `//go:build e2e`，避免慢测拖累日常。
3. **夹具插件不进 `extensions/plugins/`**——那里是产品插件（ADR-0016 §5）。为测异常路径而生的插件（故意崩溃、engines 不符、重复能力）一律放 `engineering/e2e/fixtures/plugins/`。
4. **E2E 不 mock 契约**——走真实 contracts/proto/协议/进程，否则测不出协议漂移。

### 语言侧规则

- TypeScript 使用仓库钉定的 pnpm，所有 workspace 必须通过 `tsc --noEmit`；运行时行为使用 Vitest，就近放置 `*.test.ts(x)`。UI 契约测试必须同时覆盖装配成功和未签名制品、不兼容契约、重复贡献等 fail-closed 路径。
- Python SDK 使用标准库 `unittest`，测试位于 `extensions/sdk/python/tests/`；涉及协议和执行驱动的行为还必须由 `engineering/e2e/` 中的真实 Python 进程测试覆盖。
- Go、TypeScript、Python 的契约测试必须消费同一份 Schema/Proto 真源，不得复制字段枚举或另造测试 DTO。

## 二、怎么写

- **命名**：`Test<被测对象>_<场景>`，如 `TestRegister_SingleDispatchRejectsDuplicate`。
- **表驱动优先**：多分支用 `cases := []struct{...}` + `t.Run(c.name, ...)`。
- **断言消息说清"期望什么、实际什么"**：`t.Fatalf("期望按 priority 降序 [high low]，实际 [%s %s]", ...)`。读失败信息就能定位，不必回看代码。
- **注释写"为什么这条不变量重要"**，而非复述代码：
  ```go
  // single 语义下同一能力 id 不得被两个插件重复提供。
  func TestRegister_SingleDispatchRejectsDuplicate(t *testing.T) { ... }
  ```
- **fail-closed 路径必须有测试**（ADR-0017 §4 的五个强制点）——只测通过路径等于没测。

## 三、怎么跑

```bash
./engineering/tools/test.sh          # 单元测试（快，日常）
./engineering/tools/test.sh --e2e    # 单元 + E2E（含跨进程真实链路）
pnpm typecheck                       # 全部 TypeScript workspace 类型检查
pnpm test:frontend                   # Frontend 内核与插件 Vitest
PYTHONPATH=extensions/sdk/python python3 -m unittest discover -s extensions/sdk/python/tests -v
```

等价于：

```bash
go test ./...                    # 单元（e2e 因 build tag 被排除）
go test -tags=e2e ./engineering/e2e/...      # E2E
```
