# 系统架构概览

> 状态：草案｜最后更新：2026-07-14
> 本文是 VastPlan 顶层架构的单一真相源；骨架细节见 [系统骨架架构](01-系统骨架架构.md)，各模块展开到 `design/` 与 `plugins/`。

## 1. 系统目标

- 面向企业级客户的**通用 LLM Agent 系统**。
- 支持**在线 Agent 开发**（在平台上编排 Agent、工具、工作流）。
- 支持**远程连接任务客户端**，将脚本/工作流下发到客户本地执行。
- 核心：**微内核 + 全层扩展点**——内核最小，功能通过第一方插件在骨架之上持续扩展，扩展面覆盖后端/Web UI/RS/RC 四层。

> 插件当前全部由本方开发（第一方、可信），暂不开放第三方；插件市场用于分发本方插件，第三方开放与安全隔离为未来议题。

## 2. 已确定的方向性决策

| 决策 | 选择 | 详见 |
|---|---|---|
| 插件化架构模型 | **微内核 + 全层扩展点**（后端/Web UI/RS/RC，一切功能皆第一方插件） | [ADR-0001](../decisions/ADR-0001-插件运行模型.md) |
| 技术栈 | **全新选型**，不受 testa 约束 | [ADR-0002](../decisions/ADR-0002-技术栈选型.md) |
| 插件装载模型 | **运行时热装**（在线安装/启停/卸载，内核不重部署） | [ADR-0003](../decisions/ADR-0003-插件装载模型.md) |
| 插件运行形态 | **独立进程 + 协议总线**（故障隔离、栈自由） | [ADR-0004](../decisions/ADR-0004-插件运行形态.md) |

## 3. 顶层分层（草图）

```
┌─────────────────────────────────────────────────────────┐
│  插件市场 (Marketplace)   分发本方插件 / 版本 / 在线安装   │
├─────────────────────────────────────────────────────────┤
│  内核 Kernel  生命周期 · 扩展点注册表 · 协议总线 · 横切服务 │
│  ↓ 内嵌于四类宿主 ↓                                        │
│  WebUI Host   │ Backend Host │  RS Host    │  RC Host      │
│  前端扩展点   │ 后端扩展点   │  RS 扩展点  │  RC 扩展点     │

└─────────────────────────────────────────────────────────┘
```

详见 [系统骨架架构](01-系统骨架架构.md)。

## 4. 四层扩展面

一个功能特性可横跨多层，各层各一份贡献，逻辑上是一个插件：

1. **后端**——工具包、Agent、权限校验器、事件汇、Hook。
2. **Web UI**——视图 slot、编辑器、命令、设置项。
3. **RS**——Runner 调度能力、执行模式、协议扩展。
4. **RC**——本地执行能力、采集器、运行时适配。
