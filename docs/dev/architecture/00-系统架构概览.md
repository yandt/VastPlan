# 系统架构概览

> 状态：草案（骨架）｜最后更新：2026-07-14
> 本文是 VastPlan 顶层架构的单一真相源。细节展开到 `design/` 与 `plugins/` 下的专篇。

## 1. 系统目标

- 面向企业级客户的**通用 LLM Agent 系统**。
- 支持**在线 Agent 开发**（在平台上编排 Agent、工具、工作流）。
- 支持**远程连接任务客户端**，将脚本/工作流下发到客户本地执行。
- 核心差异化：**高度插件化**——系统能力通过安装插件持续增强，配套**在线插件市场**（类似 VSCode）。

```
┌─────────────────────────────────────────────────────────┐
│  插件市场 (Marketplace)   在线浏览 / 安装 / 版本 / 审核    │
├─────────────────────────────────────────────────────────┤

├─────────────────────────────────────────────────────────┤
│  平台后端 (Server)        Agent 编排 / LLM 调用 / 插件后端  │

│  任务客户端 (Runner)      远程连接 / 本地执行脚本与工作流   │
│                          插件的客户端侧代码宿主            │
└─────────────────────────────────────────────────────────┘
```

## 4. 全栈插件的三个注入点

一个插件可声明性地在三处注入能力，任意组合：

1. **前端贡献点**——向工作台注入 UI（视图、面板、编辑器、设置项）。
2. **后端贡献点**——注入工具、Agent 能力、LLM 编排逻辑、API 端点，运行在隔离沙箱中。
3. **客户端贡献点**——注入下发到 Runner 本地执行的脚本/工作流。

> 三处如何通过统一的插件清单（manifest）声明、如何隔离、如何跨端通信——展开到 `plugins/`。
