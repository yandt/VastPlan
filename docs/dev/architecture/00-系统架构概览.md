# 系统架构概览

> 状态：草案｜最后更新：2026-07-14
> 本文是 VastPlan 顶层架构的单一真相源；骨架细节见 [系统骨架架构](01-系统骨架架构.md)，各模块展开到 `design/` 与 `plugins/`。

## 1. 系统目标

- 面向企业级客户的**通用 LLM Agent 系统**。
- 支持**在线 Agent 开发**（在平台上编排 Agent、工具、工作流）。
- 支持**远程连接任务客户端**，将脚本/工作流下发到客户本地执行。
- 核心：**微内核 + 全层扩展点**——内核最小，功能通过第一方插件在骨架之上持续扩展。内核分**三套**（后端/前端/客户端），后端内核可灵活组合出 backend/workspace/rs 等服务，三套内核共用同一套统一插件定义。

> 插件当前全部由本方开发（第一方、可信），暂不开放第三方；插件市场用于分发本方插件，第三方开放与安全隔离为未来议题。

## 3. 顶层分层（草图）

```
        统一插件定义（同一份清单，含 webui/backend/rc 贡献，任意组合）
             │              │                     │
   ┌─────────▼────────┐ ┌───▼──────────┐ ┌────────▼─────────┐
   │  后端内核         │ │  前端内核     │ │  客户端内核       │
   │ 组合出 backend /  │ │ (浏览器)      │ │  (客户端本地)     │
   │ workspace / rs …  │ │              │ │                  │
   └──────────────────┘ └──────────────┘ └──────────────────┘
   各内核经协议总线承载独立进程/组件的第一方插件；后端服务可合可拆
```

详见 [系统骨架架构](01-系统骨架架构.md)；内核/服务之间如何通信见 [内核间与服务间通信](02-内核间与服务间通信.md)。

## 4. 三内核 / 扩展面

三套内核共用统一插件定义；一个特性可横跨多面，逻辑上是一个插件：

1. **后端内核**（组合出 backend/workspace/rs）——工具包、Agent、权限校验器、事件汇、Hook、Runner 调度。
2. **前端内核**（浏览器）——视图 slot、编辑器、命令、设置项。
3. **客户端内核**（RC，本地）——本地执行能力、采集器、运行时适配。
