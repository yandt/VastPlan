# ADR-0004 插件运行形态：独立进程 + 协议总线

- 状态：已采纳
- 日期：2026-07-14

## 背景

插件后端代码与内核的关系有两种：同进程模块（内联注册）或独立进程/服务（RPC 通信）。这决定故障隔离强度、热装难度、以及插件能否用不同技术栈实现。

## 决策

插件以**独立进程/服务**运行，通过**稳定协议总线**（RPC/消息）与内核宿主通信。插件不与内核共享地址空间；它启动后连接宿主，声明其贡献点，宿主据此把它接入扩展点。

## 备选方案

- **模块化单体（同进程）**：性能最好、最简单（对标 testa 的 Registry 内联注册），但热装难（动态加载/卸载模块受语言限制）、故障不隔离（插件崩溃拖垮内核）、且插件被绑定到内核技术栈。综合本项目需求被否决。
- **混合**：作为局部优化保留——极高频、极轻量的内核内置能力可同进程，但对外插件一律独立进程。

## 影响

- 正面：
  - **热装自然**（ADR-0003）：装载=启动进程，卸载=停止进程。
  - **故障隔离**：插件崩溃不影响内核与其他插件。
  - **技术栈自由**：插件只需实现协议，可用与内核不同的语言（呼应 ADR-0002 栈解耦）。
  - **四层一致**：后端/Web UI/RS/RC 都用同一套"宿主 + 协议 + 插件进程"心智。
- 代价：
  - 引入进程间通信开销与协议版本管理。
  - 需要一套稳定的**插件-宿主协议**作为地基（贡献点声明、调用、事件、生命周期指令）。
- 安全隔离**非当前目标**（第一方可信）；独立进程带来的**故障隔离**才是取它的主因。
