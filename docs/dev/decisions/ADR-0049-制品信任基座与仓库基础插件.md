# ADR-0049 制品信任基座与仓库基础插件

- 状态：已采纳
- 日期：2026-07-17
- 关联：[ADR-0016 单仓与代码目录布局](ADR-0016-单仓与代码目录布局.md)、[ADR-0026 远端制品仓库与供应链信任](ADR-0026-远端制品仓库与供应链信任.md)、[ADR-0043 插件启动授权](ADR-0043-插件启动授权与签名时间边界.md)

## 背景

ADR-0016 为避免“先下载插件服务才能下载插件”的循环，把制品仓库和期望态整体设为 Backend 编译期例外。这个选择解决了早期自举，却让存储、索引、市场和远端服务实现持续进入内核发布物，也阻碍 S3、OSS、OCI 等独立演进。

如果反过来把整个仓库做成普通热装插件，普通插件将可能同时取得制品来源和“已经可信”的解释权；仓库插件本身也无法在空节点上从自己下载自己。因此需要把不可委托的信任基座与可替换的仓库能力分层。

## 决策

1. 采用“**内核信任/自举基座 + 预置仓库基础插件**”混合模型，修订 ADR-0016 §4 的“插件服务整体内置”范围。
2. Backend 内核永久保留以下最小能力：
   - Manifest/Artifact Schema、ref、SHA-256、大小、tar 根清单和法律文件绑定验证，以及文件数、单文件和展开大小资源上限；
   - 发布者证明、信任根、撤销与运行策略强制点；
   - 多制品源选择、内容寻址可信缓存、安装事务和最后可用版本；
   - 至少一个不依赖插件运行时的本地种子源；生产发行可同时内置最小 HTTPS 源。
3. 制品源 SPI 只返回未信任 `Envelope{artifact, packageBytes, proof}`。来源不能返回“已验证”标志，也不能构造 `VerifiedArtifact`；内核验证器成功后才生成该不可伪造类型，安装器只接受它。
4. 远端仓库 HTTP 服务、对象存储/OCI 适配、索引搜索、channel 晋级、复制、GC、审批、市场 API 和管理 UI 属于基础插件能力，不进入信任基座。第三方来源适配器即使被允许，也只能提供未信任 Envelope。
5. 仓库基础插件通过随 Backend 发行的**签名种子仓库/Bundle**预置：
   - 启动顺序为内核 → 种子源 → 验证并安装基础插件 → 基础插件提供远端仓库能力 → 普通插件装配；
   - 种子源只在精确 ref 不存在时尝试下一来源；读取、权限、格式或验证失败立即 fail-closed，不得静默换源；
   - 升级后保留最后可用基础插件和种子源，仓库插件崩溃不停止已安装普通插件。
6. 当前 `artifact-server` 子命令作为迁移期兼容入口继续存在。待基础插件具备 API Route 承载、配置/凭证句柄和种子发行后，再由新 ADR/实施提交移除子命令；不能先删除可用自举路径。
7. 期望态控制面不与制品仓库重新捆绑。Deployment/Assignment 仍由控制面负责，仓库只提供不可变制品，Node Agent 仍是唯一部署执行者。

## 备选方案

- 仓库整体继续内置：自举最简单，但扩大内核、存储适配与市场能力无法独立演进，否决为长期边界。
- 仓库整体作为普通插件：架构表面纯粹，但产生自举循环并允许来源自证可信，否决。
- 完全依赖外部 OCI Registry：生态成熟，但无法覆盖所有气隙/边缘部署，保留为基础插件适配器。
- 插件同时提供来源和验证器：一个被攻破的插件即可授权自己返回的字节，否决。

## 影响

- 正面：内核只保留不可自举、不可委托的信任与安装强制点，仓库产品能力可独立升级和集群化。
- 正面：来源插件被攻破时，没有受信任发布私钥仍不能生成可安装制品。
- 代价：发行流程增加签名种子仓库，并要维护基础插件最后可用版本。
- 迁移：现有本地/HTTPS 仓库成为首批内置来源实现；`artifact-server` 暂保留，后续迁移为预置基础插件。

## 实施记录（2026-07-18）

- 新增第一方 `cn.vastplan.platform.artifacts.repository` 基础插件，作为 HTTPS 制品服务的独立进程入口；它使用 `leader / leader-owned / cluster` 运行策略并提供状态工具。
- HTTP 传输层迁出 Backend 内核包至 `core/shared/go/artifactapi`。该层仅处理 TLS、认证、大小限制和协议，不包含信任结论；`pluginservice.HTTPRepositoryAdapter` 是内核保留的窄适配器，所有发布/读取仍须经过 `SignedRepository` 的签名、撤销、内容和不可变性检查。
- `artifact-server` 保留为兼容入口，并复用同一 HTTP 传输层，不再保有另一套服务实现。
- 尚未完成签名种子 Bundle、Edge API Route 与设置/凭证句柄绑定；因此兼容入口和内核种子源仍不得移除。这些是后续单独实施项，而非本次迁移完成的部分。

## 实施更新（2026-07-22）

- Node Portal Kernel 已通过固定 BFF Route 承载仓库的小载荷目录、容量、引用、生命周期、GC 与迁移操作；大字节制品和离线 Bundle 继续走独立 TLS 数据面，因此 2026-07-18 记录中的 Edge API Route 缺口已经关闭。
- Seed Inventory/LKG、确定性离线 Bundle、File Provider、File Volume A/B 迁移和可信宿主仓库自升级事务已经实现。它们不等同于随正式 Backend 发行并由生产发布密钥签署的种子 Bundle。
- 通用 Manifest 配置页、受控凭证句柄和生产发行用签名种子 Bundle 仍未完成；`artifact-server` 兼容入口和内核种子源因此继续保留。
