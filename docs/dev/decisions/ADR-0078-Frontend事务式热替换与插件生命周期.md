# ADR-0078：Frontend 事务式热替换与插件生命周期

- 状态：已接受
- 日期：2026-07-19
- 关联：[ADR-0052 前端门户内核与多 UI 设计系统插件](ADR-0052-前端门户内核与多UI设计系统插件.md)、[ADR-0062 Frontend 可信 ESM 制品与运行描述](ADR-0062-Frontend可信ESM制品与运行描述.md)、[ADR-0073 Portal 内容寻址交付快照](ADR-0073-Portal内容寻址交付快照.md)

## 背景

Portal 的前端插件是经过 RuntimeSpec 锁定、按内容摘要寻址并由浏览器复算摘要后执行的 ESM 制品。生产链路的不可变性是安全边界，但本地开发若每次修改都重新打包全部插件、发布快照并重启 Portal Edge，反馈周期过长。

简单替换一个模块 URL 或刷新页面不能解决问题：插件注册会产生页面、Slot、服务订阅和本地状态；设计系统、Shell 组合与布局又是全局单例。若缺少完整生命周期，连续更新会残留旧贡献、重复监听器或形成半新半旧的 UI。

## 决策

Frontend Kernel 增加事务式 Portal Generation（Portal 运行代）和可选的插件热替换生命周期。每次更新均创建一套不可变候选 Generation，完整执行模块摘要校验、来源校验、Portal 装配、契约校验和状态恢复；全部成功后才原子替换活动 Generation。候选失败时中止候选作用域，活动页面继续运行。

```text
源码变化
   │
   ▼
增量构建 ──失败──> 报告错误，活动 Generation 不变
   │
   ▼
摘要校验与候选装配 ──失败──> abort + dispose 候选
   │
   ▼
捕获旧状态 → 校验状态胶囊 → 候选恢复
   │                              │失败
   │                              └────> abort + dispose 候选
   ▼
原子提交新 Generation
   │
   └──> abort + dispose 旧 Generation（超时后宿主强制清理）
```

具体规则：

1. **整代切换**：设计系统、Shell 组合、布局和功能插件统一经过完整 Portal 装配。首期不原地修改注册表，也不为不同插件类别维护多套替换算法。
2. **宿主作用域**：宿主为每个 Generation 创建 `AbortController`，并通过窄生命周期上下文把只读 `AbortSignal` 交给插件。订阅、异步任务和其他副作用必须响应该信号；插件不得控制 Generation 或伪造插件身份。
3. **状态胶囊**：插件可选择实现 `capture`、`restore` 和 `dispose`。状态只能在同一插件 ID 的前后两代之间迁移，必须是可序列化 JSON，限制深度与字节数；禁止 DOM、函数、凭证、会话令牌和跨插件共享对象。没有生命周期实现的插件仍可热替换，但其插件私有状态会重置。
4. **先验证后提交**：候选模块加载、注册、状态恢复任一步失败均不得影响活动 Generation。提交后旧代清理失败只记录开发诊断；宿主仍通过 abort 和有界超时回收它能控制的资源。
5. **串行与合并**：同一 Portal 同时只允许一个替换事务。构建期间发生的新变化合并到下一代，旧候选不得晚到覆盖新候选。
6. **生产信任不降级**：生产 RuntimeSpec、内容寻址 URL、签名制品、Edge 快照和浏览器摘要复算保持不变。开发模块采用独立 descriptor 解析入口，生产解析器永不接受 `__vastplan_dev` URL。
7. **开发通道本机化**：本地 `platformdev` 才提供开发模块、Runtime 覆盖和 SSE 更新事件，并拒绝非回环访问。正式 Portal 构建不包含连接开发通道的行为。
8. **状态保持边界**：宿主持有的路由、导航位置与可序列化插件状态可跨代保持。任意 React Hook 局部状态只有插件显式放入状态胶囊时才保证迁移；不以不稳定的模块实现细节冒充通用 Fast Refresh。

## 选择理由

相较页面自动刷新，事务式 Generation 能在编译或装配失败时保留可工作的页面。相较原地增删页面/Slot，重建候选注册表直接复用冷启动校验，避免两套规则漂移。它增加了一层前端内核状态机，但这层复杂度对应真实的原子性、回滚和资源回收问题，不属于开发工具的偶然复杂度。

## 后果

- Portal Runtime 必须支持带 Generation `AbortSignal` 的注册上下文，并保留已加载模块的生命周期句柄。
- Portal Shell 必须通过稳定 React Root 提交新的 `PreparedPortal`，不能整页 reload。
- 首方插件若建立订阅或长期异步任务，必须接入生命周期信号；需要保持编辑状态的插件应提供有界 JSON 状态胶囊。
- 本地开发编排器负责监听、构建、事件广播和诊断；生产 Portal Edge 不承担源码监听职责。
- 验收必须证明：无页面刷新、成功切换、状态迁移、重复更新、编译失败、摘要错误、装配失败和 dispose 超时均有确定结果。
