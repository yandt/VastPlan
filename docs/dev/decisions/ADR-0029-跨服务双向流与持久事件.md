# ADR-0029 跨服务双向流与持久事件

- 状态：已采纳
- 日期：2026-07-16
- 关联：[ADR-0007 内核间与服务间通信模型](ADR-0007-内核间通信模型.md)、[ADR-0008 骨架技术选型对比](ADR-0008-骨架技术选型对比.md)、[ADR-0025 NATS 控制面、能力寻址与多节点调度](ADR-0025-NATS控制面寻址与多节点调度.md)、[ADR-0027 NATS 生产安全与最小权限](ADR-0027-NATS生产安全与最小权限.md)

## 背景

ADR-0025 已实现跨服务一元 request-reply 和 Core NATS 非持久事件，但大输入、日志/进度、大输出和交互调用不能靠一次性聚合 payload 模拟，否则内存上限、背压和取消都不可控。跨服务领域事件还需要服务重启后续消费、至少一次重试和发布端去重，Core NATS 无法提供这些语义。

## 决策

1. **一元与流式传输分工保持明确**。普通一元 capability 继续走 NATS request-reply；流式 capability 使用 `contracts/proto/addressing/v1` 的 gRPC 双向流 `CapabilityStream.Open`。同一个流覆盖客户端流、服务端流和双向交互，不再创建三套近似协议。
2. **能力目录发布流式端点**。流式注册记录带 `stream_endpoint`；调用方按 capability 解析健康实例并轮询选择端点。端点未发布时 fail-closed，不把流式调用回退成聚合的一元请求。
3. **帧有独立的单调序号和最终结果**。起始帧固定序号 0，携带 `CallTarget`、`CallContext` 与可选初始 payload；双方 payload 各自单调递增。输入结束与最终 `CallResult` 是显式帧，错序、错 request ID 或非法帧立即终止。
4. **背压和取消使用传输原生语义**。`Send` 直接写 gRPC/HTTP2 流，由 HTTP/2 flow-control 阻塞快生产者；不在寻址层建立无界队列。调用方 context 取消或主动 `Cancel` 会关闭 RPC，远端 handler 的 context 同步取消。
5. **生产流式端点强制 TLS 1.3 mTLS**。服务端必须配置 `RequireAndVerifyClientCert` 与 Client CA，客户端必须显式提供 transport credentials；只有本地开发和测试能显式 `Insecure`，两种模式禁止混配。
6. **持久事件使用独立 JetStream limits stream**。`VASTPLAN_EVENTS_V1` 保存 `vp.event.persist.v1.>`，默认保留 7 天、10 GiB、丢弃最旧记录。发布必须带稳定事件 ID，并映射为 JetStream message ID，在 10 分钟窗口内抑制重复写入。
7. **durable consumer 提供至少一次和断点续消费**。消费成功才显式 ACK；handler 失败延迟 NAK，默认最多投递 5 次。默认 `MaxAckPending=1` 保证单 durable 严格处理顺序。关闭本地订阅只停止拉取，不删除服务端 durable 游标。
8. **持久事件权限是最小集合**。node/runtime 只获得持久事件 subject、目标 stream INFO 与 consumer create/info/next 权限；controller 只因打开控制面句柄获得 stream INFO。stream 管理仍只属于 bootstrap。

## 备选方案

- **所有流式数据分片后发 NATS request-reply**：需要自建分片关联、窗口、背压、半关闭和取消协议，且容易形成无界缓存，拒绝。
- **流式调用先完整聚合再调用一元接口**：实现简单但不是真流式，大 payload 会放大延迟和内存峰值，拒绝。
- **所有事件都使用 JetStream**：会给瞬时 UI 通知和可丢遥测增加 durable consumer 运维成本，拒绝；调用方必须显式选择 `Publish` 或 `PublishPersistent`。
- **订阅关闭时删除 durable**：可减少服务端对象，但会丢失离线游标，与恢复语义冲突，拒绝。

## 影响

- 正面：跨服务大数据和交互调用具备真实背压、顺序校验和端到端取消；领域事件能在进程重启后继续至少一次处理，并抑制常见发布重试重复。
- 代价：流式服务需同时运营 NATS 与 gRPC mTLS 端点；至少一次语义要求 handler 仍保持幂等；严格顺序默认牺牲单 durable 吞吐。
- 边界：跨地域流量治理、流式端点证书自动轮换、死信事件处置策略和按租户 retention/配额另行设计。
