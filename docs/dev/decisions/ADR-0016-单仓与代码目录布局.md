# ADR-0016 单仓（monorepo）与代码目录布局

- 状态：已采纳
- 日期：2026-07-15
- 关联：[ADR-0003 运行时热装](ADR-0003-插件装载模型.md)、[ADR-0010 插件服务与部署编排](ADR-0010-插件服务与部署编排.md)、[ADR-0014 四内核结构](ADR-0014-四内核结构.md)、[ADR-0015 命名规范](ADR-0015-内核与贡献面命名规范.md)
- 活的目录树见《[代码目录结构](../guides/代码目录结构.md)》（本 ADR 只记决策与原则，不记会演进的细节）
- 后续细化：[ADR-0060 五大区域仓库布局与根目录收敛](ADR-0060-五大区域仓库布局与根目录收敛.md)（取代本 ADR 的具体顶层物理布局，保留 monorepo 与分层原则）

## 背景

进入编码前需定仓库策略与代码布局：四内核（backend/frontend/runner/mobile）、共享库、第一方插件、proto 契约、插件 SDK、部署编排。

## 决策

### 1. 单仓 monorepo

四内核 + 共享库 + 插件 + proto + 文档同处一仓。理由：

- **proto 是四内核共享的契约真源**（CallContext/协议），改一次要**原子地**联动所有消费者；多仓会退化为跨仓版本联动。
- **插件跨四面**（一个插件含 backend/frontend/runner/mobile 部分，ADR-0014/0015），同处一目录才符合"逻辑上是一个插件"。
- **共享客户端核心库**被 Runner 与 Mobile 共用（ADR-0014）。
- 契合文档纪律"**代码与文档同一次改动**"。
- testa 先例即单仓。

### 2. 顶层布局

`docs/` `contracts/proto/` `core/kernels/` `core/shared/` `extensions/plugins/` `extensions/sdk/` `engineering/deploy/` `engineering/tools/`——按**角色**分层，不按部署单元分。

### 3. 不设 per-service 代码目录（重要推论）

**backend / workspace / rs 不是三份代码**，而是**同一个 backend 内核二进制**部署多次 + 不同期望态 `service_role`（ADR-0003 运行时热装 + ADR-0010 编排）。故：

> **服务组合是配置，不是代码**——目录里没有 `services/backend`、`services/workspace`、`services/rs`。这是"同源不分叉"（ADR-0006）落到目录上的样子。

### 4. 插件服务编译期内置进 backend 内核（自举）

插件服务（制品仓库 + 期望态）存在**自举鸡蛋问题**：节点代理必须先有插件服务才能下载插件。故它**编译期内置**于 `core/kernels/backend/`，用 ADR-0003 早已预留的"**混合**"退路。**其余功能仍一律是插件**——此为受控的、有理由的例外，不是对"一切功能皆插件"的松动。

### 5. 插件按 id 一目录、四面同处

`extensions/plugins/<plugin-id>/{vastplan.plugin.json, backend/, frontend/, runner/, mobile/}`，面目录名 = 规范 ID（ADR-0015）。

### 6. proto 为契约单一真源

`contracts/proto/` 下定义，codegen 产出到 `core/shared/go`、`core/shared/ts`；**不允许手写重复的契约结构**。

## 备选方案

- **多仓 polyrepo**：边界硬、发布独立；但 proto 每改一次要跨仓版本联动，跨面插件被拆散，共享客户端核心重复。否决。
- **设 per-service 代码目录**：与热装 + 期望态组合矛盾，会诱导三份分叉代码。否决。
- **插件服务作为普通第一方插件**：更纯粹，但自举需额外机制（预置种子包）。否决（复杂度不划算）。

## 影响

- 正面：契约原子变更；跨面插件内聚；共享库自然复用；文档与代码同仓同改。
- 关注点：
  - 单仓多语言工具链（Go / TS / Python / 原生移动）需统一构建与 codegen 流水线（`engineering/tools/`）。
  - 仓库体量与 CI 时长增长，需按路径做增量构建/测试。
  - 内置插件服务是唯一编译期例外，须在文档中显著标注，防止例外扩散。
