# ADR-0053 门户访问策略作为独立基础插件

- 状态：已采纳
- 日期：2026-07-18

## 背景

Portal Edge 调用 Composer 必须经过 Backend 的 `permission.checker` 强制点。系统设置自举策略只适用于 `platform.settings`，不能承担门户业务授权；内核没有权限绕过通道。

## 决策

新增首方基础插件 `com.vastplan.foundation.security.portal-access-policy`。它以经验证的 `CallContext.Principal.SystemRoles` 对 `platform.portal-composer` 分别授权：草稿/提交需 `portal.compose`，审批需 `portal.approve`，发布/回滚需 `portal.publish`，读取需上述任一角色或 `portal.read`。仅 `system` 调用可执行 break-glass。Composer 对 `kernel.config.get` 与 `kernel.portal.catalog.validate` 的回调只允许其自身插件身份。

## 备选方案

扩展 bootstrap-policy 会混合设置自举与门户业务；仅管理员放行不支持职责分离。因此均不采用。

## 影响

`portal-edge` 必须将该策略作为已验证制品启动，缺失或失联时 capability 调用继续 fail-closed。
