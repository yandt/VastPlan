# ADR-0026 远端制品仓库与供应链信任

- 状态：已采纳
- 日期：2026-07-16
- 关联：[ADR-0023 插件 Schema 与可验证制品仓库](ADR-0023-插件Schema与可验证制品仓库.md)、[ADR-0024 单节点自动装配与回滚语义](ADR-0024-单节点自动装配与回滚语义.md)、[系统架构 §3](../architecture/系统架构.md)

## 背景

ADR-0023 只建立了本地不可变仓库和 SHA-256 完整性校验。摘要能够证明下载字节未变化，却不能证明“谁发布了这份代码”；若远端仓库或索引被攻破，攻击者仍可同时替换对象和摘要。Node Agent 还需要在企业内网、私有 CA 和气隙环境中运行，因此验证不能依赖在线第三方透明日志。

## 决策

1. **签名对象是完整制品证明（attestation）**。证明使用版本化 JSON，覆盖制品 ref、SHA-256、大小、对象名、完整插件清单、publisher、keyId、算法和 UTC 签署时间；算法固定为 Ed25519。签名不直接覆盖压缩包字节，而是通过证明内的 SHA-256 与字节绑定。
2. **发布者身份必须与清单绑定**。证明 `publisher` 必须等于 `vastplan.plugin.json#publisher`，服务端发布和 Node Agent 下载各自独立验证，任何一侧都不能只信任 HTTPS 或仓库返回值。
3. **信任根由部署方显式管理**。版本化信任文档按 `(publisher,keyId)` 保存 Ed25519 公钥，支持同发布者多密钥并存、NotBefore/NotAfter 轮换窗口和紧急 `revoked` 封禁。Node Agent 使用本地快照，气隙环境不依赖联网查询。
4. **远端协议使用 HTTPS 和分离 Bearer Token**。读令牌与发布令牌分离，空令牌不表示匿名访问；服务端强制 TLS，Node Agent 可附加企业私有 CA。令牌不进入命令行默认配置，工具优先从环境变量读取。
5. **仓库与客户端双重 fail-closed**。服务端只保存受信任证明；客户端先验证证明和请求 ref，再下载对象并复算摘要、大小、tar 根清单和索引绑定。缺失证明、撤销密钥、未知字段、明文 HTTP、身份不一致和对象损坏均拒绝安装。
6. **私钥只存在发布流水线**。密钥工具生成 PKCS#8 PEM，文件要求 `0600` 或更严格；Node Agent 和制品服务只持有公钥信任文档。密钥轮换通过新增 keyId、切换签署、最后撤销旧 key 三阶段完成。
7. **当前存储是单写者文件后端**。进程内发布串行化保证 ref 与证明不可变；多副本服务必须先接共享对象存储和条件写入，不能让多个文件后端实例共享目录冒充分布式事务。

## 备选方案

- **只依赖 HTTPS 和仓库账号**：传输安全不能抵抗仓库凭证或服务端被攻破，拒绝。
- **把 SHA-256 当发布者身份**：摘要没有身份语义，拒绝。
- **直接采用 OCI Registry/Cosign**：生态成熟，但会把当前最小客户端、气隙 bundle 和插件清单绑定引入较大外部运行依赖。保留未来适配器；当前先固定独立的信任契约。
- **服务端验证后客户端不再验签**：无法抵抗仓库侧篡改，拒绝。

## 影响

- 正面：远端仓库被攻破时，攻击者没有发布私钥仍不能向 Node Agent 投递可运行代码；轮换、撤销、私有 CA 和气隙信任快照均有明确边界。
- 代价：发布流水线必须安全保管私钥和同步信任文档；撤销旧密钥会使它签署的历史制品立即不可安装，需要在撤销前重签或保留已安装内容。
- 边界：透明日志、硬件密钥/KMS、OCI 适配器、SBOM 与来源构建证明、air-gap bundle 是兼容扩展，不在本 ADR 的首个实现内。
