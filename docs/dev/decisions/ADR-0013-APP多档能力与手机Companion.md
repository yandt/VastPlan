# ADR-0013 APP 内核多档能力：桌面完整 runner + 手机 Companion

- 状态：已修订——手机由"APP 内核的一个档"提升为**独立的移动 Companion 内核**，见 [ADR-0014](ADR-0014-四内核结构.md)。本篇的**能力划分（桌面完整 runner vs 手机 Companion）仍然有效**，仅承载改为两套独立内核。
- 日期：2026-07-15
- 关联：[ADR-0009 技术栈](ADR-0009-内核技术栈选型.md)、[ADR-0011 组合是通用内核能力](ADR-0011-组合是通用内核能力.md)、[ADR-0012 APP 内核运行模型](ADR-0012-APP内核运行模型.md)、[ADR-0014 四内核结构](ADR-0014-四内核结构.md)

## 背景

APP 内核（客户端，Go）目标平台是否包含手机（iOS/Android）。技术上 Go 可经 `gomobile bind` 编译到 Android AAR / iOS framework，且 ADR-0012 的预编译模型契合应用商店（禁止运行时下载可执行代码）。但手机 OS 策略与 APP 的核心行为冲突：iOS 基本禁止后台常驻/长连接、禁止本地 fork/exec 与任意解释器、禁止下载改变行为的代码、禁止自更新二进制；Android 亦大幅受限（前台服务 + Doze + Play 限制）。因此手机做不了"完整 runner"。

## 决策

**APP 内核按目标平台分能力档**（是 ADR-0011"客户端多 Profile"的具体化——手机 = 受限 target + 受限插件集）：

| 档 | 平台 | 能力 | 运行/分发 |
|---|---|---|---|
| **完整 runner** | 桌面/服务器 mac/win/linux | 常驻连接、后台任务、本地脚本/子进程执行、采集 | 预编译二进制 + 自更新（ADR-0012） |
| **Companion 伴侣端** | 手机 iOS/Android | 监控 Agent 运行、通知、**人在环审批**、触发工作流、查看结果；前台交互为主 | gomobile 编译进原生/跨端外壳；预编译合规；分发走 **App Store / 企业 MDM** |

**手机 Companion 明确不做**：后台常驻连接、本地任意脚本/子进程执行、运行时下载代码、自更新二进制——这些留给桌面/服务器完整 runner。

## 备选方案

- **手机也做完整 runner**：iOS 平台策略基本不允许（后台常驻/本地执行/自更新），不现实。否决。
- **暂不支持手机**：可行的保守选项；但企业场景"手机看/批/触发、桌面跑重活"价值明确，故现在纳入为 Companion 档。

## 影响

- 正面：手机作为伴侣端补齐"人在环审批 + 移动监控/触发"，契合企业 LLM Agent 使用场景；预编译模型（0012）天然合规商店。
- 与 0012 的**局部偏离**：手机档分发/更新走 **App Store / MDM**，而非自更新二进制（平台强制）。
- 关注点：
  - Companion 的具体贡献面/插件集（哪些 `webui`/`rc` 贡献在手机可用；重执行类不可用）。
  - 移动端与后端的通信仍走边缘入口（前台在线时），推送用系统级 Push（APNs/FCM）承接离线通知，而非常驻 NATS 连接。
  - iOS/Android 各自的后台/权限/审批交互细节，落到 Companion 专篇（后置）。
