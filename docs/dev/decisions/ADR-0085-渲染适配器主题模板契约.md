# ADR-0085 渲染适配器主题模板契约

- 状态：已采纳
- 日期：2026-07-19
- 关联：[ADR-0052](ADR-0052-前端门户内核与多UI设计系统插件.md)、[ADR-0064](ADR-0064-Portal语义组件契约与动态表单运行时.md)、[ADR-0079](ADR-0079-Frontend框架无关多语种契约.md)、[ADR-0084](ADR-0084-主题与工作台展示档位.md)

> 本决策细化 ADR-0084 的主题目录设计，并以 `themeTemplate` 取代其中早期的 `theme` 配置字段；ADR-0084 保留为当时的决策记录。

## 背景

Arco、MUI 以及未来的 UI 框架都有不同的主题实现：有的切换 CSS 属性，有的重建 Theme Provider，有的同时影响组件和 Overlay。若由 Platform Profile 直接传递框架主题对象，门户内核会被框架耦合；若只约定一个未定义语义的字符串 `theme`，则不能让配置界面可靠地发现、展示和校验可用模板。

## 决策

1. `ui.render.adapter` 必须声明 `themeTemplates` 和 `defaultThemeTemplate`。每项模板包含稳定 `id`、可本地化 `label` 与框架无关的 `scheme`（`light`、`dark` 或 `high-contrast`）。这是所有渲染适配器共同实现的契约，而不是 Arco/MUI 的私有约定。
2. Platform Profile 仅以 `renderAdapter.config.themeTemplate` 选择模板 ID。Portal Runtime 在加载任何功能插件前校验模板目录、默认模板和选择值；目录为空、ID 重复/非法或选择不存在时 fail-closed。
3. 模板描述的是用户可见的语义，不是 CSS/token 传输通道。适配器负责将其映射到框架原生能力：Arco 使用受 Shadow DOM 隔离的官方深色 CSS 属性，MUI 使用 `createTheme`/`ThemeProvider`。只有能被适配器完整实现的模板才可声明。
4. 可本地化标签用于将来的管理配置界面发现并选择模板；功能插件与布局插件均不能覆盖模板、注入框架主题对象或传递任意样式。用户级偏好若引入，也只能在当前适配器声明的目录内选择。

## 备选方案

- **每个 UI 框架自定义主题字段和切换 API**：门户、配置中心和未来适配器需要逐一适配，拒绝。
- **独立主题插件管理全部模板**：对当前单一渲染适配器的原生主题机制增加了单例、装配顺序和故障域，暂不采用；将来若出现跨适配器共享的品牌模板库，可在本契约上增加受治理的模板来源。
- **Profile 传 CSS、颜色或框架主题对象**：破坏多框架替换性、CSP/Shadow DOM 边界和配置安全，拒绝。

## 影响

- 新 UI 框架只需实现统一模板目录和 Provider 输入，门户内核无需新增框架分支。
- 模板目录是适配器对配置中心的可发现能力；模板本体仍由已验证的首方模块实现。
- 高对比度等语义只有在适配器真正完成无障碍实现后才会出现，不以“部分 token 覆盖”冒充支持。
