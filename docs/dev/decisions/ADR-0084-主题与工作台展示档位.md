# ADR-0084 主题与工作台展示档位

- 状态：已采纳
- 日期：2026-07-19
- 关联：[ADR-0052](ADR-0052-前端门户内核与多UI设计系统插件.md)、[ADR-0064](ADR-0064-Portal语义组件契约与动态表单运行时.md)、[ADR-0082](ADR-0082-前端工作台组合框架.md)、[ADR-0083](ADR-0083-前端UI分层术语与插件命名空间.md)

## 背景

企业 Portal 需要在保持跨页面一致性的前提下支持深浅色、高对比度与紧凑/标准/宽松的数据工作区。若让功能插件携带 CSS、框架主题或任意组件参数，会破坏 Arco/MUI 可替换性，并使发布配置成为样式注入入口；若把全部样式放入 Workbench，则会复制设计系统职责。

## 决策

1. `ui.render.adapter` 拥有主题目录、语义 token 与框架映射。每个适配器声明受支持的命名主题及默认主题；Platform Profile 只能以 `renderAdapter.config.theme` 选择其中之一。Portal Runtime 对未知主题 fail-closed。主题配置不接受 CSS、颜色值、URL 或框架私有参数。
2. `ui.workflow.workbench` 拥有 Pattern 的展示档位，不拥有颜色、字体、圆角或 CSS。首期 Collection 支持 `compact`、`standard`、`comfortable` 密度；Profile 以 `workbench.config.collection.defaultDensity` 和 `allowedDensities` 确定平台策略，功能页面只能声明该集合中的语义密度。
3. 配置优先级固定为：Profile 默认展示档位 → 页面声明的获准档位 → 将来的用户偏好。用户偏好只能在 Profile `allowedDensities` 内选择，不能改变主题可用集合、服务端字段、权限或发布输入。
4. Layout 继续只决定 Shell 区域位置和尺寸。功能插件继续只能依赖 Workbench SDK，不能读取 UI 框架私有 token、传递 CSS 字符串或直接构造基础组件。

## 备选方案

- **把主题塞入 Workbench**：会让 Workbench 变成第二设计系统，拒绝。
- **新增独立主题基础插件**：隔离更强，但会增加 Platform Profile 基础单例、装配与故障域；当前由 render adapter 承担更小且足够，暂不采用。
- **允许功能插件附带 CSS 或主题对象**：破坏 Shadow DOM/CSP/多框架边界，拒绝。

## 影响

- 正面：主题切换和集合密度可在线治理且能跨 Arco/MUI 保持相同语义；主题失配在装配期暴露。
- 代价：新的主题、卡片、表单和 Overlay 展示档位必须同时补齐 UI Contract、所有适配器、无障碍与 Workbench 验收，不能由单个业务插件自行增加。
