# ADR-0022 Go 模块标识使用自有域名

- 状态：已采纳
- 日期：2026-07-15
- 关联：[ADR-0016 单仓与代码目录布局](ADR-0016-单仓与代码目录布局.md)、[ADR-0019 工程规范基线](ADR-0019-工程规范基线.md)

## 背景

VastPlan 的 Go 模块标识原先以 GitHub 个人账户名为前缀。模块标识同时出现在内部包导入、Proto 的 `go_package`、代码生成参数、架构门禁和许可证扫描配置中；它是 Go 工具链的包身份，不应绑定到某一个代码托管平台或个人账户。

## 决策

Go 模块的唯一规范标识改为 `cdsoft.com.cn/VastPlan`。

- 所有内部 Go import、Proto `go_package` 与 codegen 参数必须使用该标识。
- Proto 仍是契约的唯一真源；修改 `go_package` 后必须重新生成 `core/shared/go`，不得手改生成物。
- Git remote、源代码实际托管位置和模块标识相互独立，不由本决策规定。

## 备选方案

- 保留原有 GitHub 路径：迁移成本最低，但模块身份继续绑定 GitHub 与个人账户。否决。
- 使用没有域名的短名称（如 `vastplan`）：本地可用但不具备可分发 SDK 所需的全局唯一性与标准解析路径。否决。

## 影响

- 正面：模块、SDK 和生成契约使用组织自有且稳定的身份；未来可独立选择源代码托管和私有模块分发方式。
- 关注点：现有外部 Go 消费者若存在，需把本次导入路径变更按破坏性变更处理；私有模块分发时需配置与该域名一致的解析或 `GOPRIVATE` 策略。
