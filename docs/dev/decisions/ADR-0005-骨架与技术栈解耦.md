# ADR-0005 骨架设计与技术栈解耦

- 状态：已采纳
- 日期：2026-07-14

## 背景

骨架设计可以现在就绑定具体语言/框架，也可以先在架构模式层面（扩展点、注册表、生命周期、跨层契约、协议）定清楚，把技术栈作为后续单独评估。

## 决策

**骨架先与技术栈无关地设计**。先把微内核职责、扩展点模型、插件-宿主协议、生命周期状态机、跨层契约等**架构骨干**定义清楚；具体的内核语言/框架、协议序列化格式、各层实现技术，作为后续独立 ADR 逐一评估。

由 ADR-0004（独立进程 + 协议总线）加持：协议是唯一硬约束，各组件实现栈可后定、可异构。

## 影响

- 正面：先聚焦"架构对不对"，避免过早被栈细节牵制；协议先行使异构实现成为可能。
- 关注点：骨架文档须保持在概念/契约层，凡涉及具体栈处显式标注"待 ADR"，不得夹带隐含技术假设。
