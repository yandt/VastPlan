# ADR-0024 单节点自动装配与回滚语义

- 状态：已采纳
- 日期：2026-07-16
- 关联：[ADR-0010 插件服务与部署编排](ADR-0010-插件服务与部署编排.md)、[ADR-0017 版本定义与兼容性机制](ADR-0017-版本定义与兼容性机制.md)、[ADR-0023 插件 Schema 与可验证制品仓库](ADR-0023-插件Schema与可验证制品仓库.md)、[系统架构 §3](../architecture/系统架构.md)

## 背景

ADR-0010 已确定「插件服务 + Node Agent reconcile」，ADR-0023 已提供可验证的本地制品入口。下一步需要先在不引入 NATS、多节点调度和远端身份体系的条件下，把自动下载、安装、启停、实际态与失败回滚做成真实闭环，并为后续替换控制面保留稳定边界。

需要固定的关键语义是：重复收到相同期望态不得重启；候选版本启动失败不得破坏当前实例；回滚到旧 revision 必须允许；同一 revision 被改成不同内容则必须 fail-closed。

## 决策

1. **DesiredState v1 采用正式 Draft 2020-12 JSON Schema**。当前可执行子集只接受 `service` unit、不可变插件引用 `id + version + channel`、`enabled`、精确匹配的 `placement.nodeSelector`，并固定 `replicas: 1`。`portal/app`、多副本、亲和和资源调度留到后续 Schema 版本。
2. **先实现本地单节点控制面**：期望态来自可轮询的本地 JSON 文件，制品来自 ADR-0023 本地仓库，实际态原子写入本地 JSON 文件。制品源、安装器、运行时、状态存储均通过接口注入；后续 NATS KV watch 和远端仓库替换适配器，不改 reconcile 核心。
3. **安装目录按制品 SHA-256 内容寻址**。安装器复验摘要、大小、清单身份、归档路径和 backend 可执行入口，并经临时目录原子发布。不同版本可同时存在，候选失败时无需恢复被覆盖的目录或符号链接。
4. **service unit 以完整插件组合事务式切换**：新建候选宿主，全部插件完成握手、engines 校验、贡献注册与激活后，才把当前 unit 指针切到候选并关闭旧宿主。任一候选插件失败时关闭候选，旧宿主和旧 `UnitState` 保持不变。
5. **revision 与内容摘要分工**：`ObservedRevision/Digest` 记录已观察配置；只有所有目标 unit 收敛后才推进 `AppliedRevision`。允许显式应用更小 revision 以回滚；相同 revision 对应不同 digest 则拒绝。unit 是否重启由不含 revision 的内容指纹决定，因此回滚到仍在运行的旧组合不做无意义重启。
6. **`backend/workspace/rs` 共用 backend 内核宿主**。`service_role` 表示同一 backend 二进制的不同服务组合，不是内核 ID；Node Agent 对三者使用同一宿主工厂，实际功能由插件集决定。B 阶段尚不把角色注册进 mesh，也不赋予角色隐式业务能力。
7. **本阶段不实现 drain**。当前切换保证旧实例在候选健康前不退出，但候选成功后的旧宿主关闭仍可能中断在途调用；drain、跨节点流量摘除和状态迁移单独设计。
8. **优雅退出更新实际态**：Node Agent 收到退出信号后停止本进程管理的 unit，将状态标为 `stopped` 并清除 `AppliedRevision`；异常崩溃来不及写回时，下一次启动必须结合真实运行时检查而不能只信旧文件。

## 备选方案

- **直接接 NATS KV 与 queue group**：能较早展示集群形态，但会把配置传播、寻址、调度和本地事务混在一次实现中，难以隔离失败。推迟到单节点语义经测试固定之后。
- **先改 `current` 符号链接再重启**：实现简单，但 compound unit 含多个插件，单链接不足以表达组合原子性；候选失败还需额外恢复指针。backend unit 改用并行候选宿主与内存原子切换。
- **逐插件原地热替换**：会在一个 unit 内暴露新旧插件混合状态，依赖和贡献冲突难以回滚。拒绝，unit 是最小事务边界。
- **revision 必须单调增加**：阻止直接回退旧 revision，与 ADR-0017 的回滚定义冲突。拒绝；用同 revision 内容摘要冲突检测防止静默覆盖。

## 影响

- 正面：本地已形成“声明配置 → 校验下载 → 内容寻址安装 → 真实进程激活 → 实际态上报”的闭环；幂等、失败保活、回滚和停用均有单元测试与跨进程 E2E。
- 代价：每个 service unit 当前有独立 backend 宿主，候选切换期间短暂并存两套进程；安装目录尚无垃圾回收策略。
- 边界：实际态文件是单节点报告而非集群真源；进程重启后由 reconcile 重建运行时；NATS KV、queue group、远端签名鉴权、多副本放置、drain、安装 GC 仍待后续决策。
