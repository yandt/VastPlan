# ADR-0043 插件启动授权与签名时间边界

- 状态：已采纳
- 日期：2026-07-16
- 关联：[ADR-0004 插件运行形态](ADR-0004-插件运行形态.md)、[ADR-0026 远端制品仓库与供应链信任](ADR-0026-远端制品仓库与供应链信任.md)、[ADR-0036 Backend 核心 SPI 边界](ADR-0036-Backend核心SPI边界.md)

## 背景

插件宿主原先把固定 magic cookie 当成防误接入标记，但没有验证握手携带的 `launch_token` 是否属于宿主刚启动的进程。运行进程声明的贡献只经过 Schema 校验，没有与已签名清单绑定；插件进程还继承 Backend 全部环境变量。供应链证明的 `signedAt` 由签名者自行填写，只与密钥有效期比较，已过期但未撤销的私钥可回填历史时间签署新制品。

## 决策

1. 每次插件启动都创建随机、一次性的待启动记录。握手必须原子认领有效 `launch_token`；空、未知、重复或超时令牌一律拒绝。
2. Node Agent 在安装时从已验签清单冻结插件 ID、版本、Backend 贡献和允许调用的内核服务，并把它们绑定到待启动记录。运行时声明的扩展点、ID、优先级和 descriptor 必须与冻结清单逐项一致。
3. 直接二进制演示入口可以不绑定清单贡献，但仍必须由宿主创建并注入一次性令牌；该入口不作为生产装配路径。
4. 插件子进程使用显式环境变量允许列表，默认不继承 Backend 环境。制品仓库令牌、NATS 身份、TLS 私钥和其他宿主秘密不得进入插件环境。
5. 没有可信时间戳服务时，`NotBefore/NotAfter` 同时约束当前验签时刻。过期密钥签署的历史制品必须在到期前用新密钥重签；`signedAt` 超出允许时钟偏差的未来证明拒绝。

## 备选方案

- **继续只校验固定 magic**：只能防误启动，不能认证进程。拒绝。
- **只比对插件 ID 和版本**：恶意进程仍可增加权限校验器、Hook 或工具贡献。拒绝。
- **保留完整宿主环境，由插件自律**：签名插件或构建链一旦失陷即可直接读取基础设施秘密。拒绝。
- **继续接受过期密钥的历史签名**：没有可信时间证据时无法区分历史制品和事后回填。拒绝；未来引入可信时间戳或透明日志后可新增版本化证明格式。

## 影响

- 正面：签名清单、宿主启动动作和运行进程形成同一条授权链；本机其他进程不能自行加入插件总线；插件失陷不再天然获得宿主环境秘密；密钥到期真正终止签署权限。
- 代价：插件实现和清单贡献必须严格同步；需要环境变量的插件必须由部署方显式允许；密钥到期前要完成历史制品重签。
- 兼容：协议 wire 字段不变，但握手和运行时声明校验更严格。旧插件通过正式制品重新打包并保持清单一致后可继续运行。
