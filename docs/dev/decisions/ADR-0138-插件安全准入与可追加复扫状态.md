# ADR-0138：插件安全准入与可追加复扫状态

- 状态：已采纳，分阶段实施
- 日期：2026-07-24

## 背景

当前插件包已经绑定 CycloneDX SBOM，包外来源证明也由外部 Verifier 复核；但漏洞与许可证门禁仍只存在于 VastPlan 源码仓库 CI，尚未成为每个插件制品的发布、安装和激活条件。只保存发布时扫描结果也不够：插件字节可以保持不变，漏洞数据库与企业许可证策略却会持续更新。

扫描生态不会统一为一种工具。Go、Node.js、Python、容器与企业离线环境可能分别使用 govulncheck、OSV-Scanner、Trivy、Grype、ScanCode 或商业服务。把某一种扫描器、漏洞数据库客户端或许可证解释器编译进内核，会扩大可信计算基座并让安全数据库更新绑架内核发布周期。

## 决策

采用“不可变安全准入记录 + 可追加复扫状态”的双层模型。

1. 外部 Security Assessment Provider 扫描最终插件 tar 与其中已签名绑定的 SBOM，输出统一的 `AdmissionRecord v1`。记录绑定插件 tar SHA-256、SBOM SHA-256、扫描器与数据库版本、漏洞/许可证计数、原始报告摘要、策略、有效期和 pass/fail 决策，并由独立 Ed25519 Provider key 签名。
2. 内核和仓库只实现有界、语言无关的记录解析、签名、策略和摘要验证；不执行扫描器，不下载漏洞数据库，也不解释供应商私有报告。
3. `stable` 默认必须有仍在有效期内且通过部署策略的准入记录。策略按 channel、publisher 与插件命名空间选择可信 Provider、扫描器及各风险级别上限；精确规则优先，歧义 fail-closed。
4. 准入记录是不可变 ref 的 sidecar，摘要进入 testing→stable 审批、离线 Bundle、迁移与交付证据。批准后不能替换记录来改变结论。
5. 后续复扫使用 `StatusRecord v1`。每条记录绑定原准入摘要、单调 sequence 与前一记录摘要，形成只追加链；仓库保存通过和失败结果，不能用旧的仍有效记录回滚覆盖最新失败状态。
6. Repository 与 Node Agent 各自复验。Repository 在发布和交付时执行服务端门禁；Node Agent、Portal 物化宿主、Runner 与 Mobile 在安装或激活前使用本地信任快照再次验证，不能相信仓库返回的布尔状态。
7. 无复扫记录时，仍在有效期内的准入记录可作为初始状态。存在复扫记录时只接受最新、连续且未过期的链头；最新状态为 fail、过期、签名无效、链断裂或安全策略缺失时，禁止新的安装与激活。已运行实例的强制停止由独立的撤销/应急策略决定，不由仓库直接杀进程。
8. 外部详细报告不进入内核 DTO；规范记录只保存报告 SHA-256。企业归档可以保存 SARIF、CycloneDX VEX、SPDX 或扫描器原生报告，并用摘要与准入/复扫记录对账。
9. 复扫写入使用独立的 scanner 工作负载身份和最小权限，不复用普通发布令牌或浏览器会话。传输与状态持久化属于仓库插件能力，不扩大内核 SPI。

## 方案比较

### 方案 A：只依赖源码 CI

实现最省，但无法覆盖仓库外插件、不同语言构建产物、打包后依赖变化和运行节点实际安装的制品，拒绝。

### 方案 B：每个不可变版本只保存一次扫描结果

审计简单，但漏洞数据库更新后结果会陈旧；强迫每次数据库更新都重新打插件版本也会造成无意义制品膨胀，拒绝作为完整方案。

### 方案 C：不可变准入 + 可追加复扫状态

发布事实不可改写，同时允许安全知识随时间更新；代价是多一条签名状态链和复扫调度。该复杂度直接对应供应链时效性与防回滚要求，采用。

## 语言与运行方式

协议与可信复验继续用 Go，原因是仓库和四类内核的可信边界已经是 Go，复验只需要严格 JSON、摘要、Ed25519 与策略选择，不值得增加跨进程信任跳。实际扫描 Provider 独立选型：优先按目标生态选择成熟工具，可用 Go、Rust、Python、Node.js、Java 或商业服务；默认进入受控共享扫描 Runtime 或 CI Worker，只有资源隔离、许可证或第三方信任边界要求时才使用单插件独立进程。

## 后果与实施顺序

实施按四步推进：先交付共享契约和策略验证器；再接入不可变 sidecar、stable 审批与离线迁移；随后实现只追加复扫日志、独立 scanner 身份和 Node/Portal 强制；最后补齐管理中心证据、告警和企业验收矩阵。

在完整链路落地前，文档必须明确区分“VastPlan 源码 CI 漏洞/许可证门禁”和“插件制品安全准入”，不得把前者描述成后者已经完成。
