# ADR-0018 测试布局与分层

- 状态：已采纳
- 日期：2026-07-15
- 关联：[ADR-0016 单仓与代码目录布局](ADR-0016-单仓与代码目录布局.md)、[ADR-0017 版本定义与兼容性机制](ADR-0017-版本定义与兼容性机制.md)
- 活的规则见《[测试规范](../guides/测试规范.md)》

## 背景

单仓多语言（Go / TS / Python / 原生移动，ADR-0016）需要统一的测试放置规则：测试文件放哪、如何分层、跨进程/跨组件的链路测试归谁。若不定，测试会散落且慢测会拖累日常开发。

## 决策

### 1. 按语言惯例就近放，**不**建统一 `tests/` 树

单元测试与源码**同目录**（Go：`xxx_test.go` 同包；TS：`xxx.test.ts` 就近）。

理由：Go 的同包 `_test.go` 是硬约定——能测包内私有符号、`go test ./...` 天然发现、覆盖率按包统计。强行搬进统一 `tests/` 树是与工具链作对，收益为负。**跨语言统一的是分层与命名，不是物理位置。**

### 2. 跨组件 E2E 放顶层 `e2e/`，用 build tag 隔离

跨进程/跨组件的真实链路测试（内核↔插件、多面协作）不属于任何单一包，放顶层 `e2e/`，加 `//go:build e2e`。

- 默认 `go test ./...` **不跑** E2E（保持日常快反馈）。
- CI 与本地按需 `go test -tags=e2e ./e2e/...`。

### 3. 测试夹具就近 `testdata/`；**夹具插件**放 `e2e/fixtures/`

- 数据夹具：就近 `testdata/`（Go 工具链保留目录，天然忽略）。
- **夹具插件**（为测异常路径而生的插件：故意崩溃、engines 不符、重复能力）放 `e2e/fixtures/plugins/`，**不进 `plugins/`**——后者是产品插件（ADR-0016 §5）。

### 4. E2E 不 mock 契约

E2E 必须走**真实 proto/协议/进程**，不用替身。否则测不出协议漂移与两侧版本失配——而那正是 E2E 存在的理由（ADR-0017 的 fail-closed 强制点尤需真实验证）。

### 5. 分层职责

| 层 | 测什么 | 不测什么 |
|---|---|---|
| 单元 | 单包内逻辑与边界（含私有） | 跨进程、真实 IO |
| E2E | 跨进程真实链路、fail-closed 强制点 | 细粒度分支（交给单元测试） |

## 备选方案

- **统一 `tests/` 树**：跨语言观感一致；但与 Go 工具链冲突（无法测包内私有、破坏 `./...` 发现与覆盖率归属）。否决。
- **E2E 不加 build tag**：少一个概念；但慢测会拖累每次 `go test ./...`，最终导致没人跑测试。否决。
- **E2E 用 mock 协议**：快；但等于不测协议本身。否决。

## 影响

- 正面：日常测试快（默认不含 E2E）；E2E 真实可信；夹具与产品插件边界清晰。
- 关注点：
  - E2E 需构建被测二进制，耗时较长，CI 需单独阶段。
  - build tag 易被遗忘 → 由 `tools/test.sh` 统一封装两档运行。
  - TS/Python/移动端的具体框架待各内核落地时补充（规则已定，实现随后）。
