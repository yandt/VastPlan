# ADR-0015 内核与贡献面命名规范

- 状态：已采纳
- 日期：2026-07-15
- 关联：[ADR-0014 四内核结构](ADR-0014-四内核结构.md)、[插件契约与协议](../architecture/插件契约与协议.md)

## 背景

四内核的命名此前中英混用（后端/前端 中文、Runner 英文、移动+Companion 中英混），且贡献面键 `webui / backend / rc / mobile` 中 `rc` 与内核名 Runner 不同名——工程上不一致。在写代码前统一为单一规范。

## 决策

**规范 ID（唯一真源）：全小写英文，贯穿代码 / 配置 / 命名空间 / 清单贡献面 / 扩展点前缀。**

| 显示名 | 规范 ID | 贡献面 `contributes.<id>` | 扩展点前缀 | 运行环境 |
|---|---|---|---|---|
| Backend 内核 | `backend` | `contributes.backend` | `tool.` / `permission.` / `event.` … | 服务端 |
| Frontend 内核 | `frontend` | `contributes.frontend` | `view.` / `editor.` / `command.` … | 浏览器 |
| Runner 内核 | `runner` | `contributes.runner` | `runner.` | 桌面/服务器客户端执行器 |
| Mobile 内核 | `mobile` | `contributes.mobile` | `mobile.` | 手机 iOS/Android |

**迁移**（相对早期文档）：
- `webui` → **`frontend`**（内核显示名 WebUI 内核 → Frontend 内核）。
- `rc` → **`runner`**：`contributes.rc`→`contributes.runner`，`rc.script`→`runner.script`，`rc.workflow`→`runner.workflow`，激活事件 `onRcScript`→`onRunnerScript`。
- **"RC" 仅保留为通信角色名**：在 RS↔RC 反向连接语境中，RC = Runner 作为连接到 RS 的客户端角色；不再作为内核/贡献面标识符。

**显示名规范**：四内核统一写作 "`<English>` 内核"（Backend/Frontend/Runner/Mobile 内核）；中文（后端/前端/移动/执行器）仅作行文解释性词，不作标识。

## 影响

- 正面：规范 ID 全小写英文、四面同构（backend/frontend/runner/mobile），代码/配置/契约/文档一致；消除 `rc` 与 Runner 的错位。
- 需同步：清单 `contributes` 键、扩展点名、激活事件、架构文档显示名。
- RS（Runner Service）/ RC（通信角色）与 `runner` 规范 ID 并存，语义分层清晰。
