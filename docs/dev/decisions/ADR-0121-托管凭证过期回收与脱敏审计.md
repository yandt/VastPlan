# ADR-0121 托管凭证过期回收与脱敏审计

- 状态：已采纳并实现
- 日期：2026-07-23
- 关联：[ADR-0090](ADR-0090-插件配置与托管凭证闭环.md)、[ADR-0092](ADR-0092-业务插件拥有托管凭证生命周期.md)、[ADR-0114](ADR-0114-一次性ConfigurationAuthority与委托凭证暂存.md)、[ADR-0120](ADR-0120-Service-Hot托管凭证提交与退役.md)

## 背景

托管凭证 Saga 可能因提交方崩溃、网络中断或候选被放弃而遗留 `Preparing` 与 `Aborted` 记录。永久保留前者会让无主密文持续增长；直接按创建时间删除所有状态又会破坏仍可供 readiness 使用的 `Candidate`、稳定运行的 `Active` 以及需要保留取证事实的 `Retired`。

凭证生命周期同时需要面向安全管理员的审计，但 handle、stage ID、一次性 ConfigurationAuthority、密文和 material 都不应进入浏览器、日志或普通插件响应。

## 决策

### 1. 维护职责属于凭证插件

凭证插件是托管状态与密文的唯一所有者，因此由 leader 实例在现有可信进程内运行维护循环，不再创建独立 GC 插件或进程。维护策略通过插件的 service-scope 配置提供，变更采用 restart 生效：

- `Preparing` 默认超过 24 小时自动转为 `Aborted`，并在同一次耐久提交中清空密文；
- `Aborted` 默认保留 30 天后删除记录；
- 审计事件默认保留 180 天，单租户最多 10000 条；
- 默认每小时执行一次，每租户每轮最多处理 200 条候选。

时限和批量均有运行时安全上下界，不能只依赖 Manifest JSON Schema。每轮先按租户、更新时间和 stage ID 确定顺序，再原子保存记录、审计和维护统计；保存失败必须回滚内存变化。

### 2. 可运行状态永不按时间回收

`Candidate` 与 `Active` 不参与 TTL 回收。是否放弃 Candidate 由拥有事务事实的配置协调器决定；是否退役 Active 由拥有真实 commit 事实的目标业务插件决定。`Retired` 已无密文但保留元数据用于取证，也不由该通用 TTL 删除。

GC 只处理可以安全判定为无运行价值的过期 `Preparing` 和已经完成终止的 `Aborted`。维护循环重启后从同一状态文件继续，不建立第二真相源。

### 3. 审计只投影脱敏事实

每个事件保存递增 ID、`SHA-256(handle)` 截断后的 128 位十六进制指纹、动作、状态、owner、purpose、resource、是否委托、非秘密配置候选绑定和时间。不得保存或返回：

- 原始 handle 或 stage ID；
- ConfigurationAuthority token/ID；
- Vault ciphertext、material 或秘密输入；
- 调用方可伪造的 owner/purpose。

审计查询按 tenant 隔离、倒序游标分页，既要求固定 Portal operation grant，又要求 `platform.credentials.audit` 权限；插件侧再要求已认证 `USER` caller 作为纵深防御。响应附带最近维护时间、累计自动终止/删除数量和当前状态计数。

### 4. 语言与运行形态

本能力继续使用 Go 并进入凭证插件已有可信 leader 进程。它与 Vault/KMS、持久状态、锁和后台定时器耦合，Go 在常驻资源、并发与故障处理上优于 Node/Python；另起语言 Runtime 不会带来生态收益，反而增加密文状态跨进程协调。该选择只约束本插件实现，不改变多语言插件协议。

## 备选方案

- **独立 GC 插件**：需要跨插件获得密文状态删除权，扩大安全边界并产生双写，否决。
- **只提供人工清理**：无法处理无人值守崩溃残留，且操作时更容易误删运行态，否决。
- **所有状态统一 TTL**：会删除 Candidate/Active 或破坏恢复，否决。
- **审计直接返回 handle/stage ID**：给管理面和日志带来可关联敏感句柄，否决。
- **只写普通应用日志**：日志没有租户分页、保留期、权限和结构化恢复保证，不能成为审计真相源，否决。

## 实施结果

- credentials 0.10.0 增加可配置维护循环、原子持久化、重启恢复、过期 Preparing 自动终止、Aborted 回收和有界审计保留。
- `listManagedAudit`、Node Portal Kernel 固定 BFF、TypeScript SDK 与 Workbench 审计页共同提供受权限保护的脱敏查询。
- 测试锁定 Candidate/Active/Retired 不被时间回收、租户隔离、游标分页、维护状态恢复，以及响应不含 handle、stage ID、authority、密文和明文。
