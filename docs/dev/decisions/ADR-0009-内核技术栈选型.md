# ADR-0009 内核技术栈选型

- 状态：已采纳
- 日期：2026-07-14
- 关联：[ADR-0002 技术栈全新选型](ADR-0002-技术栈选型.md)、[ADR-0004 独立进程+协议总线](ADR-0004-插件运行形态.md)、[ADR-0005 骨架与技术栈解耦](ADR-0005-骨架与技术栈解耦.md)、[ADR-0008 骨架技术选型对比](ADR-0008-骨架技术选型对比.md)

## 背景

骨架已在架构/契约层定清（ADR-0005 栈无关设计），且控制面选型已定（ADR-0008：NATS 主选、go-plugin 作独立进程宿主范式、Dapr 否决）。现在为三套内核（后端/RC/前端）落定实现语言与框架。关键输入：go-plugin 要求宿主为 Go；NATS 各语言客户端齐全；系统是 LLM 中心产品，但内核是基础设施、LLM 重逻辑归插件。

## 决策

| 内核 | 选型 | 
|---|---|
| **后端内核** | **Go** |
| **RC 客户端内核** | **Go** |
| **前端内核（WebUI）** | **TypeScript + React**，热装用 **Module Federation** |

**插件多语言**（内核 Go 不限制插件语言）：基础设施型插件用 Go；**LLM/AI 型插件用 Python**，经 gRPC/go-plugin 跨语言桥接接入；前端插件用 TypeScript/React。

### 后端内核 = Go
- **go-plugin 原生**：其"进程外 + gRPC + 崩溃隔离 + reattach 热升级"直接可用（ADR-0004/0008），宿主限 Go 的约束在此满足。
- **NATS 原生**：Go 是 NATS 一等客户端（ADR-0008 控制面）。
- **单静态二进制**利于"灵活组合"（ADR-0006）：合并/拆分部署都是同一二进制不同装配，交叉编译简单。
- **LLM 生态不在内核而在插件**：内核是插件宿主/寻址/编排的基础设施，AI 重逻辑放 Python 插件，经 gRPC 接入——既得 Go 的并发/部署优势，又保留 Python 的 AI 生态。
- **位置透明快路径（H1）自建**：Go 无 Orleans/Akka 式内建位置透明，但那层薄寻址（本地命中直调、否则远程）本就计划自研（ADR-0008），Go 完全胜任。

### RC 客户端内核 = Go
- **单个跨平台静态二进制**（macOS/Windows/Linux），客户现场分发最省事。
- **NATS 原生**：RC 靠 NATS client dial-out 穿 NAT、leaf 回连（ADR-0008 H2）。
- **与后端同为 Go**：契约、协议、NATS 接入、序列化代码**可跨后端/RC 复用**，心智统一。

### 前端内核 = React + Module Federation
- **Module Federation 热装生态最成熟**，契合运行时热装前端插件（ADR-0003）。
- **Studio 可开发模块**若走 IDE 化，可复用 **Theia**（React/TS）。
- 插件生态与人才面最广。

## 备选方案

- **后端 Python**：直接吃 LLM 生态，但高并发插件宿主弱（GIL/打包）、无 go-plugin 对等、单二进制/合并部署难。以"AI 逻辑归插件"化解后，否决。
- **后端 Rust / JVM(Kotlin) / .NET**：Rust 开发慢、宿主生态需自建；JVM/.NET 虽有 Akka-Pekko/Orleans 位置透明，但运行时更重、与 go-plugin/单二进制心智不合。均否决。
- **RC Rust**：二进制更小无 GC，但开发慢、且失去与后端同栈的代码复用。否决。
- **前端 Vue**：testa 延续、团队或更熟；但 Module Federation/Theia 生态不如 React 广。否决为主选，保留为团队熟悉度考量下的备选。

## 影响

- 正面：
  - go-plugin 从"候选"变为**可直接采用**（Go 宿主约束已满足）。
  - 后端与 RC 同栈 → **契约/协议/NATS 代码复用**，两个 Go 二进制维护心智一致。
  - LLM 能力通过 Python 插件接入，AI 生态零损失，内核保持轻。
- 关注点：
  - 需建立 **Go↔Python 插件的 gRPC 契约与脚手架**（LLM 插件的开发体验要打磨）。
  - 位置透明快路径（H1）仍需自研，落地前做小基准验证（承接 ADR-0008）。
  - 前端 React 若团队 Vue 积累较多，需评估迁移/学习成本。
  - 具体版本、构建工具链（Go 版本、Rspack/Webpack、Theia 取舍）、序列化格式等落到实现约定，另行记录。
- 本 ADR 落定语言/框架；**架构本身仍是栈无关的**（ADR-0005），协议与契约不因此绑死实现。
