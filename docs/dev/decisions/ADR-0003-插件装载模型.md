# ADR-0003 插件装载模型：运行时热装

- 状态：已采纳
- 日期：2026-07-14

## 背景

系统最初的诉求即"在线选择插件并安装"。装载模型有三种：运行时热装（在线安装/启用/卸载，内核不重部署）、编译期集成（构建时装配，上线需重部署）、或混合。这决定了"加功能是否要重部署内核"。

## 决策

采用**运行时热装**：插件可在线安装、启用、停用、卸载、升级，**无需重新部署或重启内核**。安装即"拉取插件包 → 注册清单 → 拉起插件 → 装配其贡献点到扩展点"；卸载即反向解绑并回收。

## 备选方案

- **编译期集成**：实现最简单、最稳，但没有"在线安装"体验，与核心诉求冲突。被否决。
- **混合**（内核+核心插件编译集成，业务插件热装）：作为**退路**保留——若某些底层能力实在无法热装，可编译期内置，但默认目标是全部热装。

## 影响

- 正面：功能随插件在线增减，契合产品定位。
- 关注点：要求稳定的运行时注册/解绑机制、清晰的生命周期与状态机、贡献点的动态挂载/卸载。**与 ADR-0004（独立进程）协同**——独立进程让热装的"拉起/回收"变为进程启停，显著降低动态加载难度。
