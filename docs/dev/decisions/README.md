# 架构决策记录（ADR）

本目录记录 VastPlan 的关键架构决策。ADR 是**只追加、不覆盖**的——一个决策一旦写下就不再修改；如果后来推翻它，新增一篇 ADR 并把旧篇状态标为「已废弃（被 ADR-XXXX 取代）」。

这样做的目的：让"为什么这样设计"的历史永不过期，正是文档保持有效的关键手段。

## 命名

`ADR-<四位序号>-<主题>.md`

## 每篇模板

```markdown
# ADR-XXXX 标题

- 状态：提议中 / 已采纳 / 已废弃（被 ADR-YYYY 取代）
- 日期：YYYY-MM-DD

## 背景
（面临什么问题、约束是什么）

## 决策
（选了什么）

## 备选方案
（还考虑过什么，为什么没选）

## 影响
（这个决策带来的正负后果）
```
