# ADR-0002 技术栈全新选型

- 状态：已采纳（方向）；具体选型已由 [ADR-0009](ADR-0009-内核技术栈选型.md) 定稿（后端 Go / Runner Go / 前端 React / 移动 gomobile）
- 日期：2026-07-14

## 背景

VastPlan 与已有的 testa 系统方向一致（LLM Agent + 远程客户端执行），testa 采用后端 Python、客户端 Go 的技术栈。是否沿用 testa 的技术选型，直接影响代码复用与架构自由度。

## 决策

**围绕插件化从零选型**，不受 testa 技术栈约束。理由：全栈插件模型对隔离、宿主通信、动态加载的要求，与 testa 的架构假设差异较大，沿用旧栈可能被历史包袱牵制。

具体的前端框架、后端语言/运行时、客户端语言留待专门评估后各自形成 ADR。

## 备选方案

- **沿用 testa**：复用成熟经验与部分代码，学习成本低；但受旧架构约束。
- **部分沿用**（保留客户端 Go）：折中；仍在评估范围内，不排除个别层复用 testa 的成熟实现。

## 影响

- 正面：架构自由度最大，可针对插件化做最优设计。
- 代价：无法直接复用 testa 代码，起步成本更高。
- 后续：仍会调研 testa 哪些**设计经验**（非代码）可借鉴，形成"可复用/需改造"分析。
